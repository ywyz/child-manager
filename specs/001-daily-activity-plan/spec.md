# Feature Specification: 首期一日活动计划完整闭环

**Feature Branch**: `docs`（共享规格）；`dev`（唯一实现）

**Feature Directory**: `specs/001-daily-activity-plan`

**Created**: 2026-07-12

**Status**: M0、M1 `complete` / M2 `in_progress` — #4 是 `dev` 的当前单实现验收入口

**Input**: 将 Child Manager 首期 M1–M8 作为一个完整 feature，交付工程与安全基础、
认证授权、必要设置、AI 模型与提示词、手工教案、异步 AI、固定 Word 导出和功能验收。

## Clarifications

### Session 2026-07-12

- Q: Spec Kit feature 覆盖整个首期还是单个里程碑？ → A: 整个首期使用一套 spec/plan/tasks。
- Q: 学期外日期如何关联学期与周次？ → A: 创建教案前必须存在当前学期；教案绑定当前
  学期快照，学期外日期的周次为空，但保存、AI 生成和导出继续。
- Q: 班级区域不完整时如何处理？ → A: 允许暂不完整；室内、户外 AI 栏目分别校验对应
  类别，只阻止缺少配置的栏目。
- Q: 两种工作日来源均不可用时如何记录？ → A: 使用结果 `unknown` 与来源 `unavailable`。
- Q: 七个系统默认提示词从哪里取得？ → A: 旧仓库提示词只作业务参考，必须按本项目
  PRD、变量白名单、固定 Schema 和 Word 模板重新编写。
- Q: 一键生成的多个预览如何避免互相失效？ → A: 各栏目独立采用；仅目标栏目或该次
  生成实际使用的输入发生变化时，预览才失效，无关栏目变化不影响采用。
- Q: AI 生成何时创建教案快照？ → A: 仅教师明确采用 AI 结果时创建；生成、重试、拒绝
  和失败由任务、AI 结果与审计记录追踪。
- Q: 任务投递服务暂不可用时生成请求是否已受理？ → A: 权威任务保存成功即返回任务 ID
  和等待投递状态，服务恢复后自动投递。
- Q: 一键生成是否包含反思？ → A: 不包含；晨间、晨谈、集体、室内、户外五栏全部完整
  后，教师才能显式点击生成反思，未点击不得生成。
- Q: 集体活动拆分成功后如何补新增环节？ → A: 先提示教师“尚未新增适龄环节”并允许采用
  拆分结果；只有拆分结果采用并保存后，教师才可独立生成新增环节。新增失败不回滚已采用
  的拆分，之后只需重新生成并采用新增环节。
- Q: 教师解除班级关联后如何处理既有作者？ → A: 立即撤销班级访问权，但保留当前教案
  和历史中的署名快照；署名不授予权限，也不能再把该教师新选为作者。
- Q: 七个提示词的稳定标识是什么？ → A: 使用 `daily_activity_plan.*` 命名空间：
  `morning_activity`、`morning_talk`、`group_activity_split`、`group_activity_add_step`、
  `indoor_area_game`、`afternoon_outdoor_game`、`daily_reflection`。
- Q: 一日活动反思的固定结构是什么？ → A: 使用 `highlights`、`issues`、`adjustments`
  三个字段；计数前分别做 Unicode NFKC 规范化，三个值拼接后按 Unicode 码点计数，字段名
  与导出换行不计、内容中的空格和标点计入，合计不超过 200，Word 按三行输出。
- Q: 集体活动新增环节返回什么？ → A: 只返回一个结构化新增环节和建议插入位置，系统
  负责合并并设置新增标记。
- Q: 七个提示词如何分配变量？ → A: 使用逐任务最小白名单，不共享无关变量；
  `teacher_context` 只包含教师主动填写的教学上下文，不含身份信息。
- Q: 管理员测试提示词应同步等待还是异步执行？ → A: 也作为 PostgreSQL 权威后台任务，
  API 立即返回任务 ID，页面轮询或恢复结果。
- Q: 集体活动尚无 AI 新增环节时是否算“完整”并可生成反思？ → A: 五栏完整只按当前正文
  满足固定 Schema 判断，不要求内容来自 AI 或存在 AI 新增环节；缺少新增环节只提示。
- Q: 点击生成反思时存在未保存修改怎么办？ → A: 先携带当前 `expected_version` 保存；仅
  保存成功后创建反思任务，保存失败或并发冲突不得创建。
- Q: 重新生成反思时是否把当前旧反思传给 AI？ → A: 不传；`current_plan` 只包含晨间、
  晨谈、集体、室内和户外五个上游栏目及必要展示上下文。

### Session 2026-07-13

- Q: 提示词变量占位符采用什么语法？ → A: 只支持词法
  `\{\{[ \t]*([a-z][a-z0-9_]*)[ \t]*\}\}` 的白名单纯替换，例如 `{{name}}`、
  `{{ name }}` 或花括号内水平 Tab；禁止表达式、过滤器、循环和嵌套访问，数组与对象使用稳定
  JSON 序列化后替换。
- Q: Word 导出时反思为空是否属于关键栏目缺失？ → A: 不属于。导出缺失确认只检查晨间、
  晨谈、集体、室内和户外五栏；反思为空不触发二次确认，但模板中的反思三行位置仍须保留。
- Q: 异步提示词测试排队后，草稿或模型设置变化是否影响该任务？ → A: 受理时冻结提示词
  正文/哈希、结果 Schema、所选模型的非密钥调用配置及单调 `call_config_revision`；草稿变化
  不影响已受理测试。若该档案的地址、模型名、能力或密钥在外呼前变化并递增 revision，Worker
  必须以 `prompt.configuration_changed` 零调用失败，管理员重新发起测试；不得把新密钥发送到
  旧冻结地址。外呼前仍实时校验账号权限、模型启用及当前地址安全。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 管理员完成安全初始化与必要设置 (Priority: P1)

作为首位管理员，我需要安全初始化园所、建立账号与班级关系并完成教案直接依赖的设置，
使教师能够在明确的园所和班级权限下开始工作。

**Why this priority**: 认证、权限和设置是所有教案能力的入口；缺少这些边界会让后续页面
无法证明数据属于哪个园所、班级或教师。

**Independent Test**: 从空系统创建首位管理员，再创建教师、当前学期、班级、年龄段、
教师关联和区域；以教师身份登录后只能看到关联班级，且不配置 AI 也能进入手工教案流程。

**Acceptance Scenarios**:

1. **Given** 系统尚未初始化，**When** 维护者完成一次性安全初始化，**Then** 系统只创建
   一名有效管理员且关闭重复初始化入口。
2. **Given** 管理员已登录，**When** 管理员创建教师、班级和教师—班级关联，**Then** 教师
   登录后只获得关联班级的访问能力。
3. **Given** 园所只有一名有效管理员，**When** 尝试停用该账号或移除其管理员角色，
   **Then** 系统拒绝操作并显示中文原因。
4. **Given** 管理员不是某班任课教师，**When** 管理员查看该班教案，**Then** 可以查看、
   导出、归档或恢复，但不能编辑正文、调用 AI 或维护该班区域。
5. **Given** 新建班级尚未配置区域，**When** 管理员保存班级，**Then** 班级可以保存并明确
   显示区域设置尚未完成。

**Milestone Mapping**: US1 保持一个端到端用户故事，但任务按 Roadmap 拆成两个独立门禁：
M2 只实现安全初始化、账号/角色、园所隔离、Cookie/CSRF、会话和身份审计；M3 在 M2 可信
身份上下文上实现学期、班级、教师关系、年龄段和区域设置。场景 4 的授权规则由 M2/M3
共同建立，待 M5 存在教案用例后完成端到端验收；不得为提前验收而在 M2/M3 创建教案空壳。

### User Story 2 - 教师完成纯手工教案闭环 (Priority: P2)

作为教师，我需要在不依赖 AI 或在线节假日服务的情况下创建、编辑、保存、查询、归档和
恢复本班一日活动计划，并能从历史快照恢复内容。

**Why this priority**: 手工闭环是首期最小业务价值，也是所有 AI 和导出能力的安全回退。

**Independent Test**: 使用已完成必要设置的教师账号，完成“选择班级和日期 → 创建 →
填写六大栏目 → 自动保存 → 手动保存 → 归档 → 恢复 → 恢复历史”的完整流程，全程关闭
AI 和在线节假日服务。

**Acceptance Scenarios**:

1. **Given** 当前学期已配置且同班同日无教案，**When** 教师选择关联班级和日期，
   **Then** 系统创建一份结构化教案并保存园所、班级、学期、年龄段和作者展示快照。
2. **Given** 同班同日已有教案，**When** 教师再次选择该日期，**Then** 系统打开原教案而
   不是创建第二份；已归档时以只读方式打开。
3. **Given** 活动日期在当前学期外，**When** 教师继续创建或编辑，**Then** 系统显示软提示、
   教学周次保持为空，且不阻止保存、AI 生成或导出。
4. **Given** 教师停止输入约 3 秒，**When** 自动保存成功，**Then** 页面显示保存状态、当前
   版本递增且不创建历史快照。
5. **Given** 教师显式保存，**When** 保存成功，**Then** 当前内容和作者顺序被保存并创建
   一条手动保存快照。
6. **Given** 两名同班教师基于相同版本编辑，**When** 后一个请求提交，**Then** 系统拒绝
   静默覆盖并提示刷新或比较最新内容。
7. **Given** 教案已归档，**When** 教师尝试编辑，**Then** 系统保持只读；恢复归档后才可
   继续编辑。
8. **Given** 教师恢复任一历史版本，**When** 恢复完成，**Then** 恢复前内容先被保存，
   选定历史成为新的当前版本，原历史记录保持不变。
9. **Given** 教案已经创建，**When** 客户端或内部调用尝试改变园所、班级或活动日期，
   **Then** 系统拒绝请求且三个归属字段保持原值，不能通过普通保存或 Repository 更新绕过。

### User Story 3 - 管理员配置模型与提示词 (Priority: P3)

作为管理员，我需要维护多个 AI 模型档案和七个任务级提示词的草稿、发布、历史、回滚与
测试，使教师只能使用满足能力要求的模型和已发布提示词。

**Why this priority**: AI 生成必须先有可审计、可回滚且不暴露密钥的管理基础。

**Independent Test**: 使用脱敏示例变量和本地替身，完成“新增模型档案 → 确认外部数据
风险 → 发布提示词 → 测试 → 修改草稿 → 回滚”的流程，并证明未发布内容不影响业务。

**Acceptance Scenarios**:

1. **Given** 管理员新增模型档案，**When** 保存 API Key，**Then** 页面只显示脱敏值，后续
   查看或日志中都不能取得明文。
2. **Given** 模型档案尚未完成风险确认或不具备任务所需能力，**When** 尝试启用或选择，
   **Then** 系统拒绝并说明缺少条件。
3. **Given** 系统首次可用，**When** 管理员查看提示词中心，**Then** 七个任务均存在只读
   系统默认提示词。
4. **Given** 管理员编辑草稿，**When** 尚未发布，**Then** 正式业务仍使用原已发布版本或
   系统默认版本。
5. **Given** 管理员恢复历史版本，**When** 恢复完成，**Then** 系统创建新的已发布版本而
   不修改原历史记录。
6. **Given** 提示词引用白名单外变量，**When** 保存或发布，**Then** 系统拒绝并显示中文
   错误；管理员也不能修改输出数据结构。
7. **Given** 某提示词已有 20 条测试记录，**When** 新测试完成，**Then** 只保留最近 20 条，
   且测试输出不会写入正式教案。
8. **Given** Redis 暂时不可用，**When** 管理员提交提示词测试，**Then** 系统仍返回任务 ID
   和等待投递状态，并在恢复后自动执行且不重复产生测试记录。
9. **Given** 提示词测试正在等待投递，**When** 管理员只修改原草稿，**Then** 该任务仍使用
   点击测试时冻结的提示词正文、结果 Schema 和模型调用快照。
10. **Given** 提示词测试正在等待投递，**When** 管理员修改所选模型的地址、模型名、能力或
    密钥，**Then** 档案的 `call_config_revision` 递增，旧任务以
    `prompt.configuration_changed` 零外部调用失败，并提示管理员重新发起测试。

### User Story 4 - 教师按栏目使用 AI 并保留决定权 (Priority: P4)

作为教师，我需要一键或按栏目生成集体活动以外的内容，分别预览、采用、拒绝或重试，
且 AI 结果永远不能静默覆盖我已经编辑的正文。

**Why this priority**: 栏目级 AI 是降低重复劳动的核心增量，但必须建立在手工闭环和教师
确认之上。

**Independent Test**: 使用固定 AI 替身让四个一键栏目分别成功、重试和最终失败，验证
成功栏目可独立预览，失败栏目可独立重试，只有明确采用才修改正文；再补全五个正文栏目
并由教师显式生成反思。

**Acceptance Scenarios**:

1. **Given** 教师点击一键生成，**When** 任务创建成功，**Then** 系统只生成晨间活动、
   晨间谈话、室内区域游戏和下午户外游戏，不包含集体活动或反思。
2. **Given** 多个栏目同时生成，**When** 其中一个失败，**Then** 成功栏目仍可预览和采用，
   失败栏目可单独重试。
3. **Given** 室内区域为空但户外区域有效，**When** 一键生成，**Then** 只阻止室内区域
   栏目，户外和其他栏目继续。
4. **Given** 生成结果已进入预览，**When** 教师关闭预览或选择保留原内容，**Then** 当前
   正文保持不变。
5. **Given** 教师采用预览前只有无关栏目发生变化，**When** 提交采用，**Then** 系统允许
   安全采用；目标栏目或本次生成实际使用的输入发生变化时，系统拒绝且不覆盖新内容。
6. **Given** 发生可恢复网络、限流、服务端或结构校验错误，**When** 系统自动重试，
   **Then** 总模型调用不超过三次并显示重试状态。
7. **Given** 发生认证、余额、权限、模型不存在或配置错误，**When** 任务失败，**Then**
   系统不自动重试，保留原内容并显示稳定中文错误。
8. **Given** 晨间、晨谈、集体、室内和户外五栏全部完整，**When** 教师显式点击生成反思，
   **Then** 系统只基于点击时已保存的完整教案生成反思；任一栏不完整或教师未点击时不得
   创建反思任务。
9. **Given** 五栏均由教师手工填写且集体活动没有 AI 新增环节，**When** 当前正文满足固定
   Schema，**Then** 反思生成可用；系统可以提示尚无 AI 新增环节，但不得把它作为门禁。
10. **Given** 五栏有尚未保存的页面修改，**When** 教师点击生成反思，**Then** 系统先按
    当前版本保存；只有保存成功才创建任务，保存失败或并发冲突时任务数保持不变。
11. **Given** 当前已有一份反思，**When** 教师显式重新生成，**Then** 外部 AI 输入只包含
    五个上游栏目及必要展示上下文，不包含当前旧反思。
12. **Given** 点击生成反思时页面含未保存修改，**When** 保存、五栏完整性校验、任务或
    pending 结果占位插入任一步失败，**Then** 正文、版本、任务和结果全部回滚；成功时只提交
    一次正文版本更新且不建快照。反思结果按 NFKC 后三个字段值的 Unicode 码点总数校验，
    超过 200 时不得形成预览。

### User Story 5 - 教师处理集体活动原始教案 (Priority: P5)

作为教师，我需要粘贴文本或上传 `.docx` 原始教案，让系统拆分、补全并新增一个适龄
环节，再由我预览和决定是否采用。

**Why this priority**: 集体活动来源于教师已有教案，其处理规则和红字来源标记不同于其他
自动生成栏目，必须单独验收。

**Independent Test**: 使用无真实身份信息的文本和 `.docx` 样本，验证文本提取、临时文件
清理、结构化拆分、缺失字段补全、新增环节标记、预览采用和取消标记。

**Acceptance Scenarios**:

1. **Given** 教师粘贴文本或上传合法 `.docx`，**When** 解析成功，**Then** 系统先展示提取
   文本供确认，长期只保存文本、原文件名、时间和内容哈希。
2. **Given** 上传完成、解析失败或处理超时，**When** 流程结束，**Then** 临时上传文件均被
   清理，不保留可下载原附件。
3. **Given** 教师确认原始文本，**When** 拆分成功，**Then** 结果包含主题、目标、准备、
   重点、难点和活动过程；原文缺少重点或难点时各补充一句。
4. **Given** 拆分成功，**When** 教师查看并采用预览，**Then** 系统明确提示尚未新增适龄
   环节；采用保存后，教师可独立创建新增环节任务，若该任务失败，只需以后重新生成新增环节，
   已采用拆分保持不变。
5. **Given** 新增环节任务完成，**When** 教师采用，**Then** 活动过程中新增一个适龄环节，
   以结构化标记保存且不插入“AI 新增”等字样，并创建本次采用快照。
6. **Given** 教师修改新增环节正文，**When** 保存，**Then** 新增标记仍保留；只有显式
   取消标记后才变为普通内容。

### User Story 6 - 教师导出并重新下载固定 Word (Priority: P6)

作为有权限的教师或管理员，我需要将指定版本的教案严格填入固定 Word 模板，并从导出
历史重新下载每一次独立结果。

**Why this priority**: 固定 Word 是园所现有工作交付物；模板错位、红字范围错误或同名覆盖
都会使前面的在线编辑失去价值。

**Independent Test**: 使用虚构园所、班级和教师数据生成两次同一教案导出，验证模板原件
哈希未变、字段和样式正确、两份历史互不覆盖且越权下载被拒绝。

**Acceptance Scenarios**:

1. **Given** 当前内容尚未保存且栏目完整或用户已确认缺失，**When** 用户发起导出，
   **Then** 系统以当前版本无快照保存，并在同一事务冻结导出内容后创建任务；保存、冻结或
   任务创建失败时全部回滚且禁止创建导出。
2. **Given** 晨间、晨谈、集体、室内和户外五个关键栏目之一存在缺失且尚未确认，**When**
   用户导出，**Then** 系统无副作用地列出缺失项并要求再次确认，不保存正文、不消耗幂等
   key、不创建导出/任务，也不自动调用 AI；反思为空不进入缺失集合且不触发确认。
3. **Given** 模板存在且哈希匹配，**When** 导出完成，**Then** 园所、学期、周次、日期、
   班级、作者和六大栏目进入固定位置，表格、字体、字号、段落和换行保持。
4. **Given** 集体活动仅部分环节带新增标记，**When** 导出，**Then** 只有这些环节正文为
   红色，其他 AI 内容保持模板原样。
5. **Given** 同一教案重复导出且班级名含空格、兼容字符或文件名非法字符，**When** 两次
   均成功，**Then** 系统保留两条内部标识不同的记录和副本；教师可见名称均严格为
   `一日活动计划_{清理后班级名}_{YYYY-MM-DD}.docx`，不含路径或非法字符。
6. **Given** 历史副本缺失，**When** 用户重新下载，**Then** 系统显示明确中文错误且不
   静默重新生成旧文件。
7. **Given** 用户无该园所或班级访问权，**When** 请求下载，**Then** 系统拒绝且不暴露
   服务器路径。

### User Story 7 - 管理员审计关键操作，教师获得可降级服务 (Priority: P7)

作为管理员，我需要查看不含敏感正文的关键审计事实；作为教师，我需要在 AI、任务投递、
后台处理、在线节假日或 Word 失败时获得明确反馈，并继续所有不依赖该故障的手工工作。

**Why this priority**: 审计和降级是系统可安全运营的完成条件，不能在功能页面完成后再
补做。

**Independent Test**: 注入各外部边界故障，验证关键操作均有脱敏审计或诊断关联，手工
正文不丢失，未受影响的流程保持可用。

**Acceptance Scenarios**:

1. **Given** 用户执行登录、权限变更、模型启停、提示词发布、AI 采用、归档或导出，
   **When** 操作结束，**Then** 系统记录操作者、时间、资源、结果和关联标识，不记录密钥
   或完整正文。
2. **Given** AI、Redis 或 Worker 不可用，**When** 教师继续操作，**Then** 数据库可用时
   创建、保存、归档、恢复和已有导出下载仍可使用；已提交的后台请求保持已受理并等待恢复，
   不得以 503 改报未受理。
3. **Given** 在线和本地工作日来源均不可用，**When** 教师选择日期，**Then** 系统显示
   “无法确认是否为工作日”，记录 `unknown/unavailable`，且不阻止主流程。
4. **Given** Word 生成失败，**When** 任务结束，**Then** 已保存教案不受影响，失败记录
   不指向可下载半成品。
5. **Given** 导出存储或模板边界不可用，**When** 教师操作，**Then** 新导出或依赖该存储的
   下载可以明确失败，但手工教案和其他已验证可用的边界不得被连带标为不可用。

### Edge Cases

- 重复执行首次初始化时，系统不得创建第二个首位管理员或覆盖已有凭证。
- 用户名规范化后重复、手机号重复或手机号为空时，唯一性规则必须保持一致且可理解。
- 停用账号、撤销会话或解除班级关联后，已有页面不能继续使用旧权限完成新请求。
- 解除班级关联后，既有教案和历史仍显示原作者姓名，但该署名不授予访问权，且该教师
  不能再被新选为作者。
- 当前学期不存在时，不允许创建教案，并引导管理员先完成学期设置。
- 活动日期在当前学期前后时，周次为空；页面和导出前提示，但其他流程继续。
- 本地与在线工作日来源结论冲突时，系统不得静默选择且应保留可诊断的最小来源元数据。
- 同班同日并发创建、归档后再次创建、以及两个相同版本并发保存都不能产生重复或覆盖。
- 自动保存失败时，页面内容必须保留，用户可以显式重试保存。
- 教案包含未知内容版本时，系统必须保留原数据并阻止编辑、AI 和导出。
- 班级对应区域为空、全部停用或名称规范化后重复时，系统只阻止相应 AI 栏目。
- 默认模型被停用、密钥缺失、能力不匹配或限流时，手工流程保持可用。
- 提示词草稿引用未知变量、历史版本被恢复或系统默认版本被编辑时，生命周期规则不得破坏。
- 一键生成重复投递、页面重载、后台处理中断或恢复时，模型调用和预览结果不得重复。
- AI 预览过期、被拒绝、目标栏目或实际生成输入已改变时，不得采用或覆盖当前正文；
  无关栏目变化不得让预览失效。
- `.docx` 扩展名伪装、文件过大、损坏、无可提取文本或含路径字符时，必须拒绝并清理临时文件。
- 模板缺失、哈希不符、结构不符、写入中断或副本存储不可用时，不得标记导出成功。
- 文件名中的班级名称含非法字符时，教师可见名称必须安全清理且内部存储键仍保持唯一。
- 日志、审计、任务消息和错误响应中出现密码、令牌、API Key 或完整正文时视为安全失败。

## Requirements *(mandatory)*

### Functional Requirements

#### 身份、权限与园所边界

- **FR-001**: 系统必须只提供交互式 CLI 完成首位管理员初始化，终端密码不得回显；园所、
  角色种子、管理员和角色分配必须在一个事务中创建。重复运行只能报告已初始化，不得修改
  既有数据；不得开放公众注册、Web 初始化或从环境变量自动创建管理员。
- **FR-002**: 用户必须能够使用用户名或已登记手机号加密码登录、退出和修改本人密码；
  Access Token 有效期 15 分钟，Refresh Token 族绝对有效期 7 天且每次刷新轮换；退出、
  密码修改、管理员重置、账号停用和 Refresh 重放必须撤销相应会话，重放撤销整个 token family。
- **FR-003**: 管理员必须能够创建管理员或教师账号、启停账号、重置密码、分配角色，且
  系统必须阻止停用或移除最后一个有效管理员。
- **FR-004**: 所有读取和状态变更必须依据服务端确认的园所、账号状态、角色和班级关系
  授权；客户端提供的园所或角色信息不得扩大权限。后台 AI/提示词测试在每次外部调用前
  必须按请求人重新验证账号、当前角色/班级、教案归档状态和模型可用性；撤权或停用后必须
  在零外部调用下不可重试失败，采用时仍再次鉴权。
- **FR-005**: 教师只能访问本人当前关联班级；管理员可查看、导出、归档和恢复全园教案，
  但只有同时作为对应班级教师时才能编辑正文、调用 AI 或维护区域。
- **FR-006**: 系统必须支持教师与班级多对多关联、每班最多一名主班教师，以及按明确
  顺序维护教案的多名编写者。解除关联必须立即撤权但保留既有署名快照；历史署名不产生
  权限，且已解除关联教师不能被新选为作者。

#### 首期必要设置

- **FR-007**: 管理员必须能够维护园所名称；首期业务时区固定为 `Asia/Shanghai`。
- **FR-008**: 管理员必须能够维护学期名称、起止日期、有效状态和唯一当前学期；起始日期
  不得晚于结束日期，有效学期日期范围不得重叠。
- **FR-009**: 创建教案前必须存在当前学期；教案必须保存创建时的园所、班级、学期、
  年龄段和作者展示快照。
- **FR-010**: 管理员必须能够维护班级、托班/小班/中班/大班年龄段、教师关联和主班教师；
  首期不提供自定义年龄段。
- **FR-011**: 任课教师必须能够分别维护本人关联班级的室内与户外有序区域；空白名称无效，
  同类名称规范化后不得重复，班级允许暂时没有区域。
- **FR-012**: 每个依赖区域的 AI 栏目必须只校验自身类别至少有一个启用区域；缺少配置
  不能阻止手工编辑或不依赖该类别的其他 AI 栏目。

#### AI 模型与提示词

- **FR-013**: 管理员必须能够维护多个供应商中立的模型档案，包括地址、密钥、模型名、
  能力、并发/限流、默认状态和启用状态。
- **FR-014**: 模型密钥必须使用带完整性保护、上下文绑定和版本轮换能力的服务端加密保存，
  且永不回显明文；档案首次启用前必须由管理员确认外部服务数据处理风险并记录确认人和时间。
- **FR-015**: 任务只能使用已启用、具备所需能力且密钥配置完整的指定或默认模型档案。
- **FR-016**: 系统必须提供晨间活动、晨间谈话、集体活动拆分、集体活动新增环节、室内
  区域游戏、下午户外游戏和一日活动反思七个稳定任务及只读默认提示词。
  七个标识依次为 `daily_activity_plan.morning_activity`、
  `daily_activity_plan.morning_talk`、`daily_activity_plan.group_activity_split`、
  `daily_activity_plan.group_activity_add_step`、`daily_activity_plan.indoor_area_game`、
  `daily_activity_plan.afternoon_outdoor_game` 和 `daily_activity_plan.daily_reflection`。
- **FR-017**: 默认提示词可以参考旧仓库的业务表达，但必须按本项目的变量白名单、固定
  结果结构、格式、数据最小化和 Word 规则重新编写；旧内容不得改变当前需求。
- **FR-018**: 管理员必须能够创建和编辑单一草稿、发布新版本、查看不可变历史，并将
  历史内容恢复为新的已发布版本；正式生成只能读取已发布自定义版本或系统默认版本。
- **FR-019**: 每个提示词必须使用固定变量白名单和固定结果结构；白名单外变量不得保存或
  发布，管理员不得通过提示词改变业务数据结构。
  - 晨间活动与晨间谈话：`plan_date`、`weekday_text`、`teaching_week_text`、`season`、
    `class_name`、`age_group_name`、`teacher_context`。
  - 集体活动拆分：`source_text`、`age_group_name`、`teacher_context`。
  - 集体活动新增环节：`group_activity`、`age_group_name`、`teacher_context`。
  - 室内区域游戏：晨间公共变量及 `indoor_areas`。
  - 下午户外游戏：晨间公共变量及 `outdoor_areas`。
  - 一日活动反思：`plan_date`、`class_name`、`age_group_name`、`current_plan`。
  占位符词法固定为 `\{\{[ \t]*([a-z][a-z0-9_]*)[ \t]*\}\}`：接受无空白或花括号内
  仅含 ASCII 空格/水平 Tab 的形式，标识符必须以 ASCII 小写字母开头，后续仅可为小写字母、
  数字或下划线；拒绝换行、其他 Unicode 空白、大小写标识符、前导下划线、点号、下标、
  未闭合占位符及白名单外变量。渲染只做一次纯替换，不得执行表达式、过滤器、循环、函数或
  嵌套字段访问。数组与对象必须按稳定 JSON 序列化，变量值中的占位符外观文本不得递归渲染。
  `teacher_context` 只能包含教师主动填写的教学上下文；学期外 `teaching_week_text` 可为空；
  反思的 `current_plan` 只含五个上游栏目和必要展示上下文，不含当前旧反思。
- **FR-020**: 管理员必须能够使用脱敏示例变量测试草稿或历史版本；测试必须作为可恢复的
  后台任务执行并立即返回任务 ID，结果不得进入正式教案。每个提示词只保留最近 20 条测试
  记录并允许管理员清空；清理只能删除已完成记录。创建时必须串行化同一提示词的计数，若
  已有 20 条未完成记录则返回 409 `prompt.too_many_active_tests` 且不创建任务或记录；重复
  幂等请求不得产生第二条任务或测试记录。任务和运行记录必须在同一事务保存经目标输入
  Schema 校验的不可变实际变量/哈希、提示词正文/哈希、结果 Schema，以及所选模型的 API
  地址、模型名、能力和单调 `call_config_revision` 非密钥调用配置快照；不得保存 API Key 或
  密文副本。地址、模型名、能力或密钥变化必须原子递增档案 revision。Worker 只能按任务 ID
  从权威记录读取冻结上下文；外呼前若当前 revision 与冻结值不同，必须以
  `prompt.configuration_changed` 零调用失败，不得混用新密钥与旧地址；revision 相同时才读取
  当前密钥，并实时校验请求人权限、模型档案启用及当前地址安全。公开响应只返回已提供变量名
  和“值已脱敏”标志，不返回完整
  输入、提示词正文或模型地址；完整上下文不得进入 Redis、日志或审计。幂等命中必须发生在
  最近 20 条清理之前；若原运行已被清理，重放仍返回原任务且关联运行标识为空，不得重新
  创建运行或再次调用模型。

#### 手工教案、日期与历史

- **FR-021**: 教案首页必须提供日历和列表视图，并支持按班级、日期范围、编写教师和归档
  状态筛选；首期不提供全文搜索。
- **FR-022**: 同一园所、班级和活动日期只能存在一份教案，归档记录仍占用该唯一位置；
  选择已有日期必须打开原教案。
- **FR-023**: 教案创建后不得修改所属园所、班级或活动日期；任何客户端字段、批量更新或
  内部 Repository 调用的变更尝试都必须被拒绝且不改变已有值。
- **FR-024**: 活动日期在当前学期内时，学期开始日期所在自然周为第一周，此后每到周一
  周次加一并显示中文周次。
- **FR-025**: 活动日期在当前学期外时必须软提示，教学周次及其显示文本必须为空；这不
  阻止保存、AI 生成或用户确认后的导出。
- **FR-026**: 系统必须按活动日期推断星期日期文本和固定四季，不得接入天气服务或向 AI
  提供虚构实时天气。
- **FR-027**: 工作日判断必须本地来源优先、在线来源补充并使用有界超时与分类缓存；两种
  来源均不可用时记录 `result=unknown/source=unavailable`；来源冲突时采用本地结论、记录
  `source=combined` 和最小摘要。非工作日、冲突或未知都只能软提示。
- **FR-028**: 教案正文必须保存晨间活动、晨间谈话、集体活动、室内区域游戏、下午户外
  游戏和一日活动反思六大结构化栏目，并验证编号、换行、标点和选择范围。反思必须包含
  `highlights`、`issues`、`adjustments` 三个字段；计数前分别做 Unicode NFKC 规范化，
  三个值拼接后按 Unicode 码点计数，字段名与导出换行不计、内容中的空格和标点计入，合计
  不得超过 200，并按三行导出。
- **FR-029**: 系统必须提供显式保存和停止输入约 3 秒后的自动保存，并显示“保存中”、
  “已保存”或“保存失败”。
- **FR-030**: 所有可变教案更新必须使用版本号检测并发；版本不匹配时必须拒绝静默覆盖。
- **FR-031**: 自动保存只更新当前内容；手动保存、教师采用仍有效的 AI 预览、归档、
  恢复归档和历史恢复必须创建带原因的完整快照。AI 生成、重新生成、重试、拒绝和失败
  本身不得创建教案快照，只由任务、AI 结果和审计记录追踪。快照原因码固定为：
  `manual_save`、`ai_adopted`、`archive`、`unarchive`、`before_restore`、`restored`。
  恢复历史版本时，覆盖当前内容前必须先以 `before_restore` 保存当前内容；复制选定历史
  成为新的当前版本后，再以 `restored` 保存恢复完成后的内容。具体恢复行为同时遵循 FR-032。
- **FR-032**: 恢复历史时必须先保存恢复前内容，再把选定历史复制为新的当前版本；历史
  记录不得移动、覆盖或删除。
- **FR-033**: 教案不得增加草稿、完成、审核或删除状态；只使用归档时间表示归档，归档后
  默认只读，恢复归档后才能编辑。

#### 异步 AI 生成与采用

- **FR-034**: 没有 AI 配置或 AI 不可用时，教师必须仍能手工填写、保存、归档和导出。
- **FR-035**: 一键生成只能创建晨间活动、晨间谈话、室内区域游戏和下午户外游戏四个
  独立栏目任务，不得包含集体活动或一日活动反思。
- **FR-036**: 每个 AI 栏目必须支持独立生成和重新生成；一键任务中的部分失败不得回滚
  成功栏目，用户必须能够只重试失败栏目。通用显式重试只接受最终失败的 AI 执行任务，不
  接受 `ai.batch` 父任务、提示词测试或 Word 导出；后两者必须通过各自创建端点表达新意图。
- **FR-037**: 权威任务保存成功后，即使任务投递服务暂不可用，系统也必须在 2 秒内返回
  可查询任务标识与“等待投递”状态，并在服务恢复后自动投递；不得因提交后的 Redis 故障
  返回 503。一键请求必须在同一事务创建一个不调用模型的 `ai.batch` 父任务和恰好四个
  同园子任务，父任务的展示状态必须实时由子任务派生并标识部分失败，不得套用或回写普通
  执行任务状态迁移。显式重试创建新 AI 任务和唯一 pending 结果占位，以同园
  `retry_of_job_id` 关联原失败任务，并复制原任务冻结的栏目基线、实际输入、模型、提示词和
  结果 Schema；父子、重试和结果关系必须由含 `kindergarten_id` 的组合外键保证。页面还必须
  显示排队、生成、重试、等待确认、已采用、已保留和最终失败状态；重复投递或恢复不得产生
  重复业务结果。
- **FR-038**: 页面重载后必须能恢复未完成任务状态，不得依赖仅存在于页面内存的计时器。
- **FR-039**: 网络、超时、限流、服务端和结构校验错误最多自动重试两次，总模型调用最多
  三次，并使用有界退避且尊重服务端限流提示；认证、余额、权限、模型不存在和配置错误
  不得自动重试。
- **FR-040**: 每次模型输出必须先通过固定结构校验并保存为预览；生成、重试或最终失败
  均不得直接修改当前教案。
- **FR-041**: 只有教师明确采用且权限有效、目标栏目及本次生成实际使用的输入均未改变时，
  才能创建快照并写入正文。无关栏目变化不得使预览失效；拒绝、关闭、过期或相关输入冲突
  时原内容必须保持。任务创建时的 `teacher_context` 是该预览不可变输入快照；采用时复用
  此快照并只重读其他可变服务端输入，页面后来为新生成修改 context 不使旧预览失效，若要
  使用新 context 必须重新生成；采用请求只携带最新 `expected_version`。
- **FR-042**: 每次生成必须记录所用模型、提示词版本、结果结构、耗时、状态和脱敏错误，
  但不得记录密钥或不必要的完整正文。
- **FR-043**: 外部 AI 请求只能包含当前任务必要的日期、周次、季节、班级、年龄段、区域、
  教师上下文和对应正文，默认排除账号、手机号和幼儿身份。

#### 集体活动导入与生成

- **FR-044**: 集体活动必须由教师先粘贴纯文本或上传 `.docx`；首期不得接受 PDF、图片
  OCR 或其他文件格式。
- **FR-045**: `.docx` 压缩文件不得超过 10 MiB、展开不得超过 50 MiB、ZIP 条目不得超过
  1000、提取文本不得超过 200,000 字符；系统必须同时校验扩展名、MIME、ZIP 签名与 OOXML
  结构，拒绝宏、外部关系、路径穿越和嵌套压缩，展示提取文本供确认，并在成功、失败或
  超时后清理临时文件；长期只保存文本、清理后的文件名、时间和哈希。
- **FR-046**: 集体活动拆分结果必须包含主题、目标、准备、重点、难点和活动过程；原文
  缺少重点或难点时分别补写一句。
- **FR-047**: 系统必须独立新增一个适龄活动环节，使用结构化新增标记且不得写入“新增”
  文案。AI 只返回单一结构化环节和建议插入位置，系统负责合并并设置 `is_ai_added=true`；
  教师编辑正文不自动取消标记，并必须提供显式取消操作。
- **FR-048**: 集体活动拆分与新增环节必须分别形成可追踪预览。拆分成功后系统必须提示
  “尚未新增适龄环节”并允许教师先采用；只有拆分结果采用并保存后才可创建新增环节任务，
  该任务只读取当时已保存的当前集体活动。新增失败不得回滚已采用拆分，之后允许只重新生成、
  预览和采用新增环节；每次采用分别创建快照。

#### 固定 Word 导出与历史

- **FR-049**: 在晨间、晨谈、集体、室内和户外五个关键栏目完整或用户确认其缺失后，发起
  Word 导出必须以 `expected_version` 无快照保存当前内容，并在同一事务创建导出记录和任务；
  导出记录必须复制保存后的不可变上下文、正文、正文 Schema 版本和正文哈希。保存、复制或
  任务创建任一步失败必须全部回滚。反思为空不属于关键栏目缺失，不触发二次确认。
- **FR-050**: 导出前必须只对上述五个关键栏目列出缺失项并要求用户确认；未确认时必须无
  副作用返回冲突，不保存正文、不消耗幂等 key、不创建导出或任务。反思为空时仍按模板保留
  三行固定位置并输出空内容。缺失只能软提示，不得自动调用 AI，也不得在用户确认后继续
  阻止导出。
- **FR-051**: 导出必须使用 `templates/teacherplan/teacherplan.docx` 的副本并校验模板
  标识、文件名和 SHA-256；模板缺失或不匹配时必须失败，不得覆盖原件或重建近似模板。
- **FR-052**: 导出必须保持模板的表格、固定标签、字体、字号、段落和换行，并按模板说明
  填入园所、学期月份范围、周次、日期、班级、有序作者和六大栏目。
- **FR-053**: 只有仍带新增标记的集体活动新增环节正文可以使用红色字体；其他 AI 内容
  不得整体标红，也不得额外加入“AI 新增”等文字。
- **FR-054**: 教师可见文件名必须为 `一日活动计划_{班级}_{YYYY-MM-DD}.docx`。班级名段
  必须先做 NFKC 规范化，将控制字符及 `/\:*?"<>|` 替换为 `_`、折叠连续 `_`、去除首尾
  空格/点/下划线，清理后为空时使用 `班级`；内部唯一存储标识必须防止同名覆盖。
- **FR-055**: 每次导出必须保留独立记录和受保护副本，记录教案版本、模板哈希、文件哈希、
  操作者和时间；后台生成只能读取导出记录内的不可变上下文/正文快照，不得重读后来变化的
  当前教案；归档或修改教案不得删除历史导出。
- **FR-056**: 下载和重新下载必须再次校验园所、班级和角色权限且不得暴露服务器路径；
  历史副本缺失时必须报错，不得静默重建。

#### 安全、审计、降级与质量

- **FR-057**: 密码、会话令牌、模型密钥和主密钥不得以明文进入代码库、业务数据、消息、
  日志、异常、审计或测试快照。
- **FR-058**: 系统必须记录登录及账号权限变更、设置变更、模型档案启停、提示词发布/恢复、
  AI 创建/重试/结果采用、手动保存、归档/恢复、历史恢复、导出和历史下载等审计事件。
- **FR-059**: 审计只保存必要元数据和关联标识，不保存密钥或完整正文；审计事件至少保留
  一年。
- **FR-060**: 用户可见错误必须使用简体中文并提供稳定问题类型；跨同步请求和后台任务的
  诊断信息必须可关联且保持脱敏，列表不得无界返回数据。PostgreSQL 不可用使用
  `database.unavailable`，完成请求所必需的服务端本地配置缺失或无效使用
  `configuration.unavailable`，不得用前者掩盖配置错误。整体 `/health/ready` 只有在
  PostgreSQL 或所有核心请求都依赖的全局安全配置不可用时才返回 503；功能专属配置或固定
  资源只让健康分项 degraded，并在实际依赖端点受理前返回 `configuration.unavailable`。
- **FR-061**: AI、在线节假日、消息投递、Worker、模板或导出存储失败时，系统必须保留已
  保存正文、给出明确状态，并保持所有不依赖该故障边界的手工能力。Redis/Worker/AI 故障
  不得使数据库手工流程或已有可读导出失效；模板/导出存储故障只允许影响新导出及实际依赖
  该存储的下载，不得把整个 API 标记为不可用；它们在整体就绪检查中只能报告 degraded。
- **FR-062**: 系统必须支持最多 100 个账号、30 个并发在线会话和至少 100 个排队任务。
- **FR-063**: 普通页面和列表操作 95% 必须在 2 秒内得到结果，保存操作 95% 必须在 1 秒内
  完成，AI 任务必须在 2 秒内创建并返回任务标识。
- **FR-064**: 首期界面必须使用简体中文和角色化侧边栏，主导航为教案、班级区域、系统
  设置、审计记录；教案使用六栏目纵向单页编辑、栏目内 AI 预览和详情内导出历史。优先
  支持桌面浏览器和平板，手机端只要求登录、浏览和简单修改；核心流程按 WCAG 2.2 AA
  验证键盘、焦点、标签/错误关联、对比度、非纯颜色状态和触控尺寸，不宣称第三方整体认证。
- **FR-065**: 常规自动化验收不得访问真实或付费 AI、在线节假日接口或其他外部网络；
  时间、AI、节假日、任务和 Word 边界必须使用固定输入或替身。
- **FR-066**: 首期功能完成前不得把生产访问网络、反向代理、生产拓扑、生产密钥托管、
  备份任务或发布流程纳入当前实现或验收；这些事项必须在功能验收后另行复审。
- **FR-067**: 晨间活动、晨间谈话、集体活动、室内区域游戏和下午户外游戏五栏全部完整
  后，系统才可启用反思生成；反思必须由教师显式触发。完整性只按五栏当前正文是否满足
  固定 Schema 判断，不要求内容由 AI 生成或集体活动含 AI 新增环节。触发用例必须先按
  `expected_version` 保存当前页面内容；保存、完整性校验和任务插入必须处于同一数据库
  事务，并同时创建唯一 pending AI 结果占位，冻结目标栏目基线、五栏实际输入及哈希、模型、
  提示词和结果 Schema；任一步失败都回滚正文、版本、任务与结果占位，成功的预保存只递增
  一次版本且不创建教案快照。`current_plan` 只含五个上游栏目及必要展示上下文，不含当前旧
  反思；未点击不得自动创建任务。
- **FR-068**: 新密码必须为 15–128 个字符，允许 Unicode、空格和粘贴，不强制字符组合或
  周期轮换，并使用本地常见/泄露密码清单拒绝弱密码。密码必须使用可升级的内存困难哈希
  保存，禁止明文或可逆存储。
- **FR-069**: 同一规范化账号 15 分钟内失败 5 次后必须启用 1–60 秒指数延迟；同一来源
  15 分钟内失败超过 30 次必须返回 429。不得自动永久锁定；不存在、密码错误和停用账号
  使用通用提示，成功、失败和限流都写脱敏审计。来源地址不得信任浏览器提供的 Forwarded/
  X-Forwarded-For/内部头；只有 API 直连对端为显式配置的回环 Web BFF 时才信任由 BFF 按
  浏览器 socket peer 重建的内部地址，否则使用 API 看到的 socket peer。
- **FR-070**: 认证 Cookie 必须使用 Secure、HttpOnly、SameSite=Lax；状态变更必须校验
  Origin/Referer，并验证签名双提交 CSRF Cookie 与 `X-CSRF-Token`。只有显式开发配置且
  Web/API 均绑定回环地址时可将 Secure 设为 false，非回环组合必须拒绝启动。
- **FR-071**: 模型地址默认只能使用公网 HTTPS，不得跟随重定向；系统必须阻止 DNS 重绑定、
  本机、链路本地、云元数据和未授权私网访问。私网或回环只能由服务端显式 allowlist 开启。
- **FR-072**: 自动/手动保存、归档、恢复归档、历史恢复、AI 采用和导出必须校验
  `expected_version`；AI 生成、提示词测试、显式重试和 Word 导出必须使用
  `Idempotency-Key`。幂等作用域固定为园所、操作者、HTTP 方法、规范化路由模板和 key；同作用域
  的请求摘要必须包含规范化后的实际路径参数、有语义的查询参数及规范化 JSON 请求体；摘要
  相同返回原任务及原受理语义，摘要不同返回冲突。除反思在创建任务事务内预保存外，一键和
  其他单栏生成必须由 Web 先以当前版本完成无快照自动保存；保存失败或冲突不得请求生成，
  生成 API 只接受保存后的最新 `expected_version`，不得以请求中的完整页面正文绕过保存。
  明确新重试必须使用新 key、只允许关联同园原失败 AI 执行任务，并用 `retry_of_job_id` 保留
  任务谱系。

### Key Entities

- **幼儿园**: 单个运行实例服务的园所，包含名称、业务时区和有效状态。
- **用户与角色**: 管理员或教师身份、登录凭证、有效状态和园所归属；同一用户可同时具有
  两类角色。
- **班级与教师关联**: 班级、年龄段、多名任课教师及至多一名主班教师的关系。
- **学期**: 名称、起止日期、当前状态和有效状态；为教案提供快照和周次上下文。
- **班级区域**: 按班级、室内/户外类别和顺序维护的可启停区域。
- **AI 模型档案**: 模型连接、密钥密文、能力、单调调用配置 revision、限流、默认状态、
  风险确认和启用状态。
- **提示词定义与版本**: 七个稳定任务、变量白名单、能力要求、固定结果结构、系统默认、
  自定义草稿和不可变发布历史。
- **一日活动反思**: 教案中的结构化反思，由活动亮点、存在问题和调整策略三部分组成；
  三个值分别 NFKC 规范化后拼接并按 Unicode 码点计算，总长度不超过 200。
- **提示词测试记录**: 仅供 Worker 使用的不可变实际示例输入/哈希、提示词正文/哈希、结果
  Schema、含 `call_config_revision` 的模型非密钥调用配置快照，对外只显示变量名与值已脱敏
  标志，以及状态、耗时和短期测试结果；不保存 API Key 快照。
- **一日活动计划**: 园所、班级、活动日期、上下文快照、六大结构化栏目、版本和归档信息。
- **教案编写者**: 教案与有序教师姓名快照的关联。
- **教案历史快照**: 特定原因下保存的完整上下文和正文，不可修改或删除。
- **集体活动来源**: 粘贴或 `.docx` 提取后的原始文本、文件名、时间和哈希，不含上传附件。
- **后台任务**: 执行任务的权威状态、调用次数、租约、方法/路径作用域幂等、
  `retry_of_job_id` 谱系和错误摘要；父子与重试关系不得跨园。`ai.batch` 只保存聚合身份，
  展示状态由四个子任务派生。
- **AI 生成预览**: 与任务、教案版本、栏目、模型和提示词关联的短期结构化候选结果。
- **Word 导出记录**: 每次独立导出的教案版本、模板与文件哈希、内部存储标识、状态和历史。
- **工作日缓存**: 某日期的工作日结论、来源、最小诊断信息和有效期。
- **审计事件**: 关键业务操作的操作者、资源、结果、时间和关联标识，不含敏感正文。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在关闭 AI、在线节假日服务和外部网络的验收环境中，教师仍能完成创建、
  六栏手工编辑、保存、归档、恢复和经确认后的 Word 导出完整流程。
- **SC-002**: 所有未登录、跨园、未关联班级和普通管理员正文编辑的负向验收请求均被
  拒绝，不出现一次越权数据返回或状态变更。
- **SC-003**: 同班同日并发创建、相同版本并发更新和重复后台投递的验收中，重复教案、
  静默覆盖、重复快照和重复导出成功记录均为 0。
- **SC-004**: 七个任务均存在可用的只读默认提示词；草稿、发布、历史、回滚和最近 20 条
  测试记录的验收场景全部通过；每条异步提示词测试在延迟投递和进程恢复后仍只使用受理时
  冻结的实际输入、提示词正文、结果 Schema 和模型非密钥调用配置。草稿变化不改变已受理
  任务；模型地址、名称、能力或密钥变化时，旧任务的外部调用数为 0，并以稳定配置变更错误
  结束。
- **SC-005**: AI 一键生成任一栏目失败时，其他成功栏目 100% 可独立预览和采用；任何未
  明确采用的结果都不改变当前正文。
- **SC-006**: 可恢复 AI 错误的每个任务总调用次数不超过 3；不可重试错误的自动重试次数
  为 0。
- **SC-007**: Word 回归样本的表格数量与结构、固定文本、字段位置、中文字体、字号、段落、
  换行和红字范围全部符合模板；原模板 SHA-256 始终为
  `72ee26e7cb8f510a11bc303b7a967c2a375fe436b5c8a72822ee9ccbfe235043`。
- **SC-008**: 系统在 100 个账号、30 个并发会话和 100 个排队任务的验收规模下，普通页面
  与列表操作至少 95% 在 2 秒内完成，保存至少 95% 在 1 秒内完成，AI 任务在 2 秒内返回
  任务标识。
- **SC-009**: 密钥、令牌和完整敏感正文在代码库、消息、日志、异常、审计和测试快照的
  自动扫描中暴露数量为 0。
- **SC-010**: PRD 第 18 节所有未延后的验收项均有自动化或可重复人工证据；无未解释的
  跳过、放宽断言或把生产部署延后项伪装成已完成。
- **SC-011**: 缺失或伪造 CSRF、错误 Origin、过期/撤销/重放 Refresh Token、非回环
  `Secure=false`、弱密码及登录限流的负向矩阵全部被拒绝，账号枚举差异为 0。
- **SC-012**: PostgreSQL 成功而 Redis 故障时，AI 和提示词测试请求仍在 2 秒内返回原任务
  ID；Redis 恢复后在一个待投递扫描周期及调度容差内执行，重复任务结果数为 0。
- **SC-013**: 每次一键请求恰好创建一个 `ai.batch` 父任务和四个栏目子任务；父任务不调用
  模型、不投递且响应只由四个子任务准确派生全部成功、部分失败和全部失败。未点击反思时反思任务数为 0；AI 生成、
  重试、失败和拒绝产生的教案快照数为 0，每次成功采用恰好产生一条。
- **SC-014**: 超过任一上传、展开、ZIP 条目或文本上限，以及含宏、外部关系、路径穿越、
  嵌套压缩或伪装 OOXML 的全部测试样本均被拒绝，测试临时文件残留数为 0。
- **SC-015**: 所有列表、错误、版本冲突和幂等场景通过 OpenAPI 契约测试；同一园所、
  操作者、方法、规范化路由模板和 `Idempotency-Key` 的相同请求重放不会新增任务、模型调用、
  快照、提示词测试或导出记录；实际 path/query/body 摘要不同返回冲突，跨资源不得错误重放；
  允许的 failed AI 显式重试新任务可追溯到同园原任务并复用原冻结输入。
- **SC-016**: 固定时钟下，工作日确认/未知/来源冲突分别遵循 24 小时、5 分钟、1 小时
  缓存；在线调用在 5 秒总超时内降级，所有情况都不阻止主流程。
- **SC-017**: 登录、必要设置、创建/编辑/保存、AI 预览采用和导出核心流程通过既定自动
  可访问性扫描与完整键盘人工清单，不存在阻断级 WCAG 2.2 AA 缺陷。

## Assumptions

- 本 feature 覆盖整个首期一日活动计划闭环，并以 Roadmap M1–M8 作为依赖实施阶段；M9
  生产安全与部署复审不属于本 feature。
- 当前 `main` 只有共享文档和模板；本规格不表示任何业务代码、迁移或测试已经存在。
- 一个运行实例首期只服务一所幼儿园，但园所隔离必须从数据、授权和验收层面完整保留。
- 当前学期是创建教案的必需设置；学期外日期仍绑定当前学期快照，周次为空。
- 班级允许暂时缺少区域；对应类别的 AI 栏目在至少有一个启用区域前不可生成。
- 系统默认提示词会参考旧仓库内容后按当前文档重写；旧仓库不是需求或架构事实来源。
- 真实 AI 服务、在线节假日服务和生产基础设施不作为常规测试或首期功能验收的依赖。
- Word 模板只有一个已确认版本；只有出现第二个真实模板需求时才重新评估模板系统。

## Dependencies

- 产品行为依赖 `docs/PRD/lesson-management.md`，架构和数据边界依赖已接受 ADR、
  `docs/design/system-architecture.md`、`docs/design/data-model.md` 与
  `docs/design/database-schema.md`。
- Word 字段、格式和样式依赖 `templates/teacherplan/teacherplan.docx` 及其系统说明。
- 七个系统默认提示词的重写需要能够读取旧仓库相应提示词作为参考，但最终内容必须独立
  接受本规格和当前模板的评审。
- M6 AI 异步生成必须同时等待模型/提示词基础和手工教案闭环完成；Word 导出在异步任务
  与手工内容稳定后实施。

- 教案提交、审核、批注、园长审批、删除、复制、另存日期或创建后修改活动日期。
- 天气服务、天气标签、PDF/图片 OCR 导入、PDF 导出、在线打印排版和全文搜索。
- 游戏观察、一对一倾听、照片上传、视觉识别以及对象存储的表、页面和部署。
- 通用模板引擎、模板上传、在线模板编辑或模板市场。
- 提示词审批、多环境发布、A/B 测试或年龄段专属提示词版本。

  密钥托管和发布流程。
