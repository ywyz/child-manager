# Feature Specification: 密码与 TOTP 备用登录

**Specification Branch**: `docs`

**Implementation Target**: `dev`（仅在本规格确认、Issue 固定 docs 提交后）

**Created**: 2026-07-23

**Status**: 已确认 — 等待固定 docs 提交与 M3A Issue

**Input**: 在通行密钥之外增加由密码与基于时间的一次性密码（TOTP）共同组成的独立备用
登录，以降低用户更换设备时无法进入系统的风险；不得加入密码单独登录、短信验证码登录或
TOTP 单独登录。

## Clarifications

### Session 2026-07-23

- Q: 新验证方式成功后获得什么权限？ → A: 作为独立备用登录，可直接建立普通业务会话。
- Q: 备用登录采用什么组合？ → A: 密码与 TOTP 两项缺一不可。
- Q: 哪些账号必须配置？ → A: 管理员强制配置；教师可选并持续提示。
- Q: 备用会话可以执行哪些安全操作？ → A: 普通业务可用；再次完成密码与 TOTP 验证后
  只能新增通行密钥，不能删除旧凭据或修改其他高风险身份设置。
- Q: 密码或 TOTP 丢失后如何恢复？ → A: 仍有通行密钥时通过 WebAuthn 重新验证重设备用
  因素；所有通行密钥也不可用时继续使用离线恢复码加人工核验。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 管理员建立备用登录 (Priority: P1)

作为管理员，我需要在首次进入业务系统前设置密码并绑定 TOTP，使通行密钥所在设备不可用
时仍能用两个独立因素登录。

**Why this priority**: 管理员账号具有全园管理权限，也是最后管理员恢复的关键主体；如果
备用因素未预先绑定，设备丢失后不能临时补建。

**Independent Test**: 新管理员完成通行密钥注册和人工激活后，必须设置合格密码并成功验证
一个 TOTP 才能进入业务页面；密码、TOTP 任一缺失或错误均不能建立会话。

**Acceptance Scenarios**:

1. **Given** 已激活管理员尚未配置备用登录，**When** 首次通过通行密钥认证，
   **Then** 系统只允许完成密码和 TOTP 设置，完成前不开放业务页面。
2. **Given** 管理员提交合格密码并绑定 TOTP，**When** 首个 TOTP 校验成功，
   **Then** 两项备用因素同时生效，TOTP 原始种子不再展示。
3. **Given** 教师尚未配置备用登录，**When** 通过通行密钥登录，
   **Then** 系统允许进入业务页面并持续提供可跳过的配置提示。
4. **Given** 用户只完成密码或只完成 TOTP，**When** 尝试结束设置，
   **Then** 系统不得启用备用登录。

---

### User Story 2 - 在新设备使用备用登录 (Priority: P2)

作为已配置备用登录的用户，我需要在通行密钥暂不可用时同时提交密码和 TOTP，进入普通业务
会话，并为新设备增加通行密钥。

**Why this priority**: 这是解决更换、损坏或重置设备不便的核心用户价值。

**Independent Test**: 在没有可用通行密钥的新浏览器中，密码与 TOTP 同时正确时建立普通
业务会话；用户再次验证两项备用因素后可增加新通行密钥，但不能删除旧凭据或改变其他认证
材料。

**Acceptance Scenarios**:

1. **Given** 用户已启用备用登录，**When** 账号标识、密码和当前 TOTP 全部正确，
   **Then** 系统建立与其角色相符的普通业务会话，并明确标记该会话来自备用登录。
2. **Given** 密码、TOTP 或账号标识任一错误，**When** 用户提交备用登录，
   **Then** 系统返回相同的通用失败提示，不泄露哪一项错误或账号是否存在。
3. **Given** 用户通过备用登录建立会话，**When** 未重新验证或重新验证已过期，
   **Then** 系统拒绝增加通行密钥。
4. **Given** 用户在最近五分钟内重新完成密码与 TOTP 验证，**When** 在新设备注册通行密钥，
   **Then** 系统允许新增凭据，并保留旧凭据和其他认证材料。
5. **Given** 用户只有备用登录保证级别，**When** 尝试删除或重命名通行密钥、重设密码或
   TOTP、轮换恢复码、管理账号角色、签发邀请或批准恢复，
   **Then** 系统拒绝并要求通行密钥重新验证。

---

### User Story 3 - 安全维护与恢复备用因素 (Priority: P3)

作为仍持有通行密钥的用户，我需要重设密码、重新绑定 TOTP 或关闭可选的备用登录；如果
全部通行密钥也丢失，我需要继续使用既有双条件紧急恢复，而不是依靠管理员或短信直接重置。

**Why this priority**: 密码和 TOTP 会引入新的秘密生命周期；维护与恢复如果弱于登录本身，
备用路径会成为账号接管入口。

**Independent Test**: 通行密钥重新验证可重设密码或 TOTP；教师可关闭备用登录，管理员
不能关闭；失去所有通行密钥时，密码与 TOTP 不能替代离线恢复码加人工核验来重置身份材料。

**Acceptance Scenarios**:

1. **Given** 用户持有有效通行密钥，**When** 最近完成 WebAuthn 重新验证，
   **Then** 可以重设密码或重新绑定 TOTP，旧备用会话和被替换的因素立即失效。
2. **Given** 教师启用了备用登录，**When** 通过通行密钥重新验证后选择关闭，
   **Then** 密码和 TOTP 同时失效，已有备用会话全部撤销。
3. **Given** 管理员尝试关闭备用登录，**When** 提交操作，
   **Then** 系统拒绝并说明管理员必须保留完整备用因素。
4. **Given** 用户已经失去所有通行密钥，**When** 忘记密码或失去 TOTP，
   **Then** 只能进入离线恢复码加人工核验流程；管理员、短信或单一恢复材料不能直接重设。
5. **Given** 紧急恢复完成，**When** 新通行密钥绑定成功，
   **Then** 旧密码、TOTP、通行密钥和会话全部失效；管理员在恢复后必须重新配置备用登录，
   教师可以选择是否重新配置。

### Edge Cases

- 密码正确而 TOTP 错误、TOTP 正确而密码错误、TOTP 已重放或超出允许时钟窗口时，都不得
  建立会话，并使用相同失败提示。
- 密码与 TOTP 必须在同一次备用登录意图中共同验证，不能把密码验证成功暴露为可观察的中间
  状态。
- TOTP 绑定未确认、绑定会话过期、二维码被重复使用或种子已经轮换时，旧材料不得生效。
- 多次并发绑定或重设只能有一个结果成为当前有效因素；其余请求必须失败且不能恢复旧种子。
- 备用登录被临时限流时，通行密钥登录保持可用；通行密钥失败计数与备用登录失败计数不得
  相互清零或绕过全局限制。
- 从备用会话新增通行密钥后，不自动提升既有会话的保证级别；用户必须用新通行密钥重新验证
  或重新登录。
- 教师未配置备用登录时不得创建空密码或空 TOTP 记录，也不得改变现有通行密钥登录和紧急
  恢复能力。
- 管理员角色新增、移除或账号恢复必须重新计算“管理员必须配置备用登录”的门禁。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须保留 WebAuthn 通行密钥作为首选且具备钓鱼抗性的登录方式，同时提供
  密码与 TOTP 两项共同成立的独立备用登录；不得提供密码单独登录、TOTP 单独登录、短信或
  邮件验证码登录。
- **FR-002**: 管理员账号必须启用完整备用登录后才能访问业务页面；教师可以选择启用或
  关闭，未启用时每次通行密钥登录成功后及本人安全设置页都应看到可跳过的明确提示，但
  不能被阻止使用业务功能。
- **FR-003**: 密码必须允许至少 8 个、最多 128 个 Unicode 字符及空格，允许粘贴和密码
  管理器，不设置大小写、数字或符号组合规则，不要求周期轮换；建立或修改时必须拒绝常见、
  可预期、已泄露及与账号或产品高度相关的密码。
- **FR-004**: TOTP 必须使用每账号唯一的高熵种子、六位数字和 30 秒周期；验证可接受当前
  周期及相邻一个周期的合理时钟偏差，但同一 TOTP 在其有效窗口内最多成功使用一次。
- **FR-005**: TOTP 种子只能在绑定流程中展示一次，并同时提供二维码和可复制的人工输入值；
  服务端必须以带完整性保护和账号上下文绑定的密文保存，页面、日志、异常、审计和测试快照
  均不得出现原始种子。
- **FR-006**: 备用登录必须在单次请求意图中同时验证规范化账号标识、密码和 TOTP，失败
  响应、耗时分类和审计公开字段不得区分未知账号、密码错误、TOTP 错误或因素未配置。
- **FR-007**: 备用登录成功必须建立可撤销普通业务会话，并保存认证方式和保证级别；该会话
  可以使用角色允许的普通业务功能，但不能被当作 WebAuthn 重新验证证明。
- **FR-008**: 备用会话只有在最近五分钟内再次同时验证密码和 TOTP 后才能增加新的通行
  密钥；该证明不能用于重命名或撤销通行密钥、修改密码或 TOTP、轮换恢复码、管理其他账号、
  角色、邀请、恢复批准或其他高风险身份操作。
- **FR-009**: 仍有有效通行密钥的用户必须通过最近五分钟 WebAuthn 重新验证才能建立、重设
  或关闭备用因素；教师可以关闭，管理员不得关闭。密码或 TOTP 变化必须撤销所有备用登录
  会话和未消费的备用重新验证证明。
- **FR-010**: 所有通行密钥均不可用时，密码或 TOTP 的遗失不得引入新恢复捷径；系统必须
  继续要求离线恢复码加人工核验，最后管理员继续要求两名不同责任人核验。
- **FR-011**: 紧急恢复完成必须撤销旧密码、TOTP、通行密钥、会话、邀请和旧恢复码。管理员
  使用新通行密钥首次登录后必须重新配置备用登录；教师可以跳过。
- **FR-012**: 备用登录、设置、重新验证、失败限流、因素轮换、关闭和恢复必须记录最小化
  审计事件；不得记录密码、密码摘要、TOTP、种子、二维码或完整请求体。
- **FR-013**: 备用登录必须按可信客户端来源、账号标识摘要和端点全局实施相互独立的限流
  与渐进等待；限流响应不得暴露账号是否存在，且不能阻止未受影响的通行密钥登录。
- **FR-014**: 用户必须能够查看备用登录是否已启用、最近修改时间和管理员强制状态，但不能
  查看密码摘要、TOTP 种子或足以离线验证 TOTP 的材料。
- **FR-015**: 系统必须在登录后通过本人可见的内建安全事件列表/警报通知用户备用登录启用、
  密码或 TOTP 变化、备用登录成功、新通行密钥由备用会话添加以及备用因素因恢复被撤销；
  该只读投影来自现有审计事件，至少保留并返回最近 20 条，不新增邮件/短信投递、已读状态或
  通知表，内容不得包含认证秘密。

### Key Entities

- **备用登录配置**: 账号是否满足密码与 TOTP 同时启用、是否因管理员角色被强制、最近修改
  时间及失效状态。
- **密码验证材料**: 只用于服务端验证的不可逆密码摘要、独立盐、参数版本和更新时间。
- **TOTP 认证器**: 加密种子、算法参数、最近成功时间步、绑定确认与撤销状态。
- **认证保证级别**: 会话使用通行密钥或密码加 TOTP 建立，以及最近一次相应重新验证时间。
- **备用重新验证证明**: 与用户和当前会话绑定、五分钟内有效且只能授权新增通行密钥的短时
  证明。
- **本人安全事件投影**: 从现有审计事件按当前园所和本人过滤的最近认证事件，只包含稳定
  事件代码、发生时间、认证方式和脱敏设备提示。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 管理员未完整配置密码和 TOTP 时进入业务页面的成功次数为 0；教师未配置时
  仍可通过通行密钥使用业务功能。
- **SC-002**: 密码与 TOTP 同时正确的备用登录成功率为 100%；任一因素错误、缺失、过期或
  重放时建立会话数量为 0。
- **SC-003**: 备用会话在没有最近两项重新验证时新增通行密钥的成功次数为 0；具有有效证明
  时可在五分钟内完成新设备通行密钥绑定。
- **SC-004**: 备用会话执行被禁止的身份操作时全部被拒绝，且不改变通行密钥、备用因素、
  恢复材料、账号或角色状态。
- **SC-005**: 密码或 TOTP 重设、教师关闭备用登录及紧急恢复后，旧备用会话下一次请求继续
  成功的数量为 0。
- **SC-006**: 账号枚举测试无法根据响应内容、状态或公开审计字段区分未知账号、因素未配置
  和任一因素错误。
- **SC-007**: 密码、密码摘要、TOTP、种子和二维码在代码库、URL、Cookie、日志、异常、
  审计、消息及测试快照的敏感信息扫描中暴露数量为 0。
- **SC-008**: 密码 + TOTP 备用登录及重新验证在自动化测试中不访问短信、邮件或其他外部
  网络服务。

## Assumptions

- 备用登录用于降低设备切换成本，不取代通行密钥的首选地位，也不宣称具备钓鱼抗性。
- TOTP 应用通常位于与浏览器不同的个人设备；若与通行密钥同时位于同一失陷设备，两个因素
  不能消除该设备完全失陷的风险。
- 管理员强制配置在首次通行密钥登录后的受限设置会话中完成；教师可以稍后配置。
- 当前 M3 T036–T045 和 `0004_settings.py` 不因本功能重编号；本功能在 M3 后、M4 前通过
  独立 Issue 和后续迁移实施。
- 内建安全通知复用现有 `audit_events`，不引入外部投递渠道、已读状态或新数据表。
- 不开放公众注册，不改变单次邀请、带外激活、离线恢复码加人工核验或最后管理员双人核验。

## Dependencies

- 依赖现有 WebAuthn 注册、认证、重新验证、凭据管理、会话撤销和恢复状态机。
- 依赖现有园所隔离、账号角色与三层身份端点限流。
- 实现前必须由新 GitHub Issue 固定包含本规格的 docs 提交。

## Out of Scope

- 密码单独登录、TOTP 单独登录、短信或邮件验证码登录、微信小程序 MFA。
- 管理员直接设置或重置其他用户的密码、TOTP 或通行密钥。
- 安全问题、生日、证件号等知识型验证。
- 通过备用会话删除旧通行密钥、关闭管理员备用登录或修改其他高风险身份材料。
- 生产短信、邮件、身份提供商或设备管理基础设施。
