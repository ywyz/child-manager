### 2.1 A1：修订模板说明

目标文件：`templates/teacherplan/一日活动计划系统说明.md`

修改要求：

1. 节假日接口统一为 `https://timor.tech/api/holiday/info/{YYYY-MM-DD}`。
2. 删除集体活动和反思中的 `[[字段名]]` 文本标记。
3. AI 新增的集体活动环节使用结构化字段 `is_ai_added=true`，正文不得写入“新增”等标记文字。
4. 反思字段统一为 `highlights`（活动亮点）、`issues`（存在问题）、`adjustments`（调整策略）。
5. 三个字段分别执行 Unicode NFKC 规范化后再拼接，按完整 Unicode 码点计数；空格、标点和 emoji 均计入，合计不超过 200。
6. 删除 `U+0000–U+FFFF`、超出范围可能截断或替换等 BMP 旧规则。
7. Word 导出固定映射为“活动亮点”“存在问题”“调整策略”三行，不输出内部字段名或结构标记。
8. 外部 AI 输入遵循数据最小化原则，只发送生成反思必需的五个上游栏目和最小必要上下文，默认排除教师账号、幼儿身份信息及其他无关正文。

建议写入的反思说明：

```markdown
#### 一日活动反思填写说明

1. 一日活动反思只能在前五个上游栏目满足结构化完整性后，由教师显式发起生成。
2. 系统只向外部 AI 发送生成反思必需的五个上游栏目和最小必要上下文，默认排除教师账号、幼儿身份信息及其他无关正文。
3. AI 返回 `highlights`、`issues`、`adjustments` 三个结构化字段，不使用 `[[字段名]]` 文本标记。
4. 三个字段值分别执行 Unicode NFKC 规范化后再拼接，按完整 Unicode 码点计数；空格、标点和 emoji 均计入，合计不得超过 200。
5. Word 导出依次映射为“活动亮点”“存在问题”“调整策略”三行，不输出内部字段名或结构标记。
```

快速验收：

```bash
rg -n "http://timor.tech|\\[\\[|U\\+0000|U\\+FFFF|截断|替换" templates/teacherplan/一日活动计划系统说明.md
rg -n "https://timor.tech/api/holiday/info|is_ai_added|highlights|issues|adjustments|NFKC|200|emoji" templates/teacherplan/一日活动计划系统说明.md
```

验收标准：

- 第一条检索不再命中被废弃的规则。
- 第二条检索明确命中新 HTTPS 地址、结构化字段、NFKC 和 200 限制。
- `.docx` 模板无工作区变化且 SHA-256 与阶段 0 相同。

- 执行结果无法满足某一阶段的明确验收标准。

只有同时满足以下条件，才能判定本方案完成：

1. 任务 A、B、C 的修复和验证证据完整。
2. M0-G1～M0-G6 均有可复现的通过证据。
3. 经单独授权完成历史隐私清理，M0-G7 关闭。
4. 在清理后的 `main` 恢复修改并重新验证通过。
5. 经单独授权形成、推送并核验最终 docs-only 基线，M0-G8 关闭。
6. 联合审查报告、CONTEXT、Roadmap、Spec、Plan 和 Tasks 状态一致。
7. M0 标记为 `complete`，M1 标记为 `ready`。

M1 分支和 Issue 操作属于后续 P3 启动工作，不属于 M0 修复方案已经自动授权或完成的内容。
