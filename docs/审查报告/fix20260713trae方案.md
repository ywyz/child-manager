## 二、执行步骤

### 步骤 1：修订模板说明文件

**目标文件**：`templates/teacherplan/一日活动计划系统说明.md`

**修改内容**：

1. 将节假日接口从 HTTP 改为 `https://timor.tech/api/holiday/info/{YYYY-MM-DD}`
2. 删除集体活动和反思中的 `[[字段名]]` 文本标记
3. 明确新增集体活动环节使用结构化字段 `is_ai_added=true`，正文不得写入"新增"等标记文字
4. 反思字段统一为 `highlights`（活动亮点）、`issues`（存在问题）、`adjustments`（调整策略）
5. 明确三个字段分别执行 Unicode NFKC 规范化后再拼接，按完整 Unicode 码点计数（空格、标点和 emoji 均计入），总计不超过 200
6. 删除 `U+0000–U+FFFF`、超出范围可能截断或替换的 BMP 旧规则
7. 明确 Word 导出固定为三行映射，不把内部字段名或结构标记写入正文
8. 添加 AI 数据最小化规则：系统只向外部 AI 发送生成反思必需的五个上游栏目和最小必要上下文，默认排除教师账号、幼儿身份信息及其他无关正文

**验收标准**：
- `rg` 不再命中旧 HTTP 地址、`[[...]]` 和 BMP 截断规则
- 新 HTTPS 地址、三个字段、`is_ai_added=true`、NFKC 及 200 限制均有明确命中
- `.docx` 模板不做任何修改，SHA-256 保持不变

### 步骤 3：补充系统架构 Q13

**目标文件**：`docs/design/system-architecture.md`

**修改内容**：

在幂等与恢复章节补充客户端幂等键定义：

```markdown
客户端幂等键定义如下：
- **scope**: 由园所、操作者、HTTP 方法、规范化路由模板和 key 组成
- **fingerprint**: 另含规范化实际 path、业务 query 和 canonical JSON body
- 相同 scope、key 和 fingerprint 返回原结果
- 相同 scope/key 但 fingerprint 不同返回冲突
- 内部批量子任务不伪造客户端幂等键，使用自身业务唯一约束
- 详细机器契约以 `specs/001-daily-activity-plan/contracts/openapi.yaml` 为准，跨端点语义和解释以 `specs/001-daily-activity-plan/contracts/README.md` 为准
```

**验收标准**：系统架构能够导航到契约定义，同时没有形成第二套不一致规则
