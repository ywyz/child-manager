## 3. 产品背景

幼儿园教师每天需要根据学期、班级、年龄段、活动区域和集体活动教案编写结构化一日活动计划。人工撰写耗时较长，格式容易不一致，固定 Word 表格的重复填写也会增加工作量。

本产品通过在线教案管理、栏目级 AI 辅助生成、人工确认、历史版本和固定 Word 导出，降低重复劳动，同时保证教师对最终内容拥有决定权。

## 4. 产品目标

### 4.1 首期目标

1. 教师可以在线创建、编辑、保存、归档、恢复和查询本班一日活动计划。
2. 系统可以根据设置与日期上下文生成集体活动以外的各栏目。
3. 教师提供集体活动原始教案后，系统可以完成结构化拆分并新增一个适龄活动环节。
4. 每个 AI 栏目支持独立生成和重新生成，任何生成结果都不能静默覆盖人工内容。
5. 系统可以严格按照固定模板导出 Word，仅将新增活动环节标红。
6. 系统保存导出历史和服务器副本，教师可以重新下载。
7. 系统具备园所隔离、权限控制和审计能力；生产备份与恢复在功能完成后另行验收。

### 4.2 成功标准

- 教师可以完成“设置 → 创建 → AI 辅助 → 人工确认 → 保存 → 导出”的完整闭环。
- 教案在不使用 AI 时仍可完全手工填写和导出。
- 外部 AI 或节假日服务失败时，教师已有内容不会丢失，主要手工流程仍可继续。
- 导出文档字段完整、模板结构不变、新增环节红字范围准确。
- 不同班级和园所的数据不会越权访问。

## 5. 用户与权限

### 5.1 角色

首期只有管理员和教师两类角色。

| 能力 | 管理员 | 教师 |
| --- | --- | --- |
| 管理幼儿园、学期、账号、班级 | 是 | 否 |
| 管理 AI 模型档案和提示词 | 是 | 否 |
| 维护班级室内、户外区域 | 仅在同时是该班教师时 | 本人关联班级 |
| 查看教案 | 全园 | 本人关联班级 |
| 编辑正文、调用 AI | 仅在同时是该班教师时 | 本人关联班级 |
| 导出教案 | 全园 | 本人关联班级 |
| 归档、恢复教案 | 全园 | 本人关联班级 |
| 查看审计记录 | 是 | 否 |

管理员默认不能修改教师教案正文，也不能调用 AI 重新生成。管理员只有同时被关联为该班任课教师时，才获得正文编辑能力。

### 5.2 账号规则

- 不开放公众注册。
- 系统初始化时安全创建首位管理员。
- 管理员可以创建教师或其他管理员账号。
- 用户使用用户名或手机号加密码登录。
- 用户名在园所内唯一；已填写的手机号在园所内唯一。
- 管理员可以启用、停用账号和重置密码。
- 管理员可以维护姓名、手机号、班级关联和主班教师。
- 不允许停用或删除最后一个有效管理员。
- 首期不提供账号物理删除。
- 用户可以输入当前密码后自行修改密码。
- 自助改密成功后撤销该账号其他刷新令牌，并要求当前会话重新登录。
- 忘记密码只能由管理员重置。

### 5.3 教师与班级

- 教师与班级采用多对多关系。
- 一个班级可以关联多名教师，并设置一名主班教师。
- 教师默认只能访问本人关联班级的教案。
- 同班教师可以共同编辑该班教案，系统使用乐观锁防止静默覆盖。
- 教案可以选择多名共同编写者，默认包含当前教师。
- Word 中按教案编写者的既定顺序展示姓名，不自动列出班级全部教师。

## 7. 首期必要设置

设置中心必须在教案 AI 能力之前可用。

### 7.1 幼儿园信息

- 幼儿园名称，默认“南通市崇川区樾府幼儿园”。
- 默认时区，首期固定使用 `Asia/Shanghai`。
- 园所信息由管理员维护。

### 7.2 学期管理

- 学期名称。
- 开始日期和结束日期。
- 当前学期标记。
- 开始日期不得晚于结束日期。
- 管理员维护学期。

### 7.3 班级与教师

- 班级名称。
- 年龄段：托班、小班、中班、大班。
- 年龄段数据结构允许后续扩展，但首期不提供自定义年龄段。
- 班级教师关联与主班教师。
- 管理员维护班级和关联关系。

### 7.4 班级区域

- 室内区域和户外区域分开维护。
- 班级可以暂时没有区域；室内、户外 AI 栏目分别只校验对应类别至少有一个已启用区域，缺少配置不得阻止手工编辑或不依赖该类别的 AI 栏目。
- 区域以有序列表维护，不使用逗号字符串作为编辑模型。
- 空白名称无效；同一类别内清理空白后不得重名。
- 任课教师维护本人关联班级的区域。
- AI 只能从当前班级已启用区域中选择区域和重点指导对象。

### 7.5 AI 模型档案

管理员可以维护多个模型档案：

- 档案名称。
- OpenAI 兼容 `API_BASE_URL`。
- 加密保存且不回显明文的 `API_KEY`。
- `MODEL_NAME`。
- 能力标签，例如 `text`、`vision`、`structured_output`。
- AI 任务并发上限（默认 2）和限流参数。
- 是否为系统默认档案。
- 是否启用。

提示词可以选择指定模型档案，也可以使用系统默认档案。业务代码不得依赖具体供应商 SDK 语义。

## 8. 核心业务对象

### 8.1 一日活动计划

一日活动计划至少包含：

- 园所、班级、活动日期。
- 学期快照、周次和星期日期文本。
- 年龄段快照。
- 编写者列表。
- 晨间活动。
- 晨间谈话。
- 集体活动。
- 室内区域游戏。
- 下午户外游戏。
- 一日活动反思。
- 当前内容版本号。
- 归档时间和归档操作者。
- 创建、更新和审计信息。

同一 `(kindergarten_id, class_id, plan_date)` 只能存在一份教案。该唯一规则包含已归档教案。创建前必须存在当前学期；教案保存创建时的园所、班级、学期、年龄段和作者展示快照。若活动日期在当前学期外，仍允许保存、生成和用户确认后的导出，但教学周次及其显示文本为空。

### 8.2 历史快照

快照记录创建原因、操作者、时间和完整结构化内容。以下操作创建快照：

- 教师采用仍有效的 AI 生成或重新生成预览。
- 手动保存。
- 归档。
- 恢复归档。
- 恢复历史版本。

自动保存只更新当前内容，不创建快照。

恢复历史版本时：

1. 先保存恢复前的当前内容快照。
2. 将选定历史内容复制为新的当前版本。
3. 不移动、覆盖或删除原历史记录。
4. 记录恢复操作者和时间。

### 8.3 导出记录

每次 Word 导出创建独立记录：

- 一日活动计划及其版本。
- 活动日期。
- 导出操作者和导出时间。
- 教师可见文件名。
- 服务器内部唯一存储键。
- 模板标识、模板文件名和模板 SHA-256。
- 导出状态和失败摘要。

重复导出不得覆盖旧的服务器副本或导出记录。

## 9. 日期与上下文规则

### 9.1 活动日期

- 新建时默认使用幼儿园时区的系统当前日期。
- 教师可以在创建时选择其他日期。
- 创建后活动日期不可修改。
- 选择已有日期时直接打开已有教案。
- 已有教案已归档时，以只读方式打开；恢复后才能编辑，不允许创建第二份。

### 9.2 学期与周次

- 活动日期不在当前学期时显示软提示，但不阻止保存、AI 生成或导出。
- 学期开始日期所在自然周为第一周。
- 此后每到下一个周一，教学周次递增一周。
- 周次显示为中文格式，例如 `第（一）周`、`第（十一）周`。
- Word 标题中的学期时间由完整日期范围缩减为月份范围，例如 `2026.3.5-2026.6.30` 显示为 `2026.3-2026.6`。

### 9.3 星期与日期格式

- 页面和 Word 使用 `周（四）3月4日` 格式。
- 日期计算按 `Asia/Shanghai` 执行。

### 9.4 季节

系统根据活动日期固定推断季节：

- 3–5 月：春季。
- 6–8 月：夏季。
- 9–11 月：秋季。
- 12–2 月：冬季。

系统不查询天气，也不向 AI 提供虚构的实时天气信息。

### 9.5 工作日

- 本地节假日库是主要判断来源。
- 在线节假日 API 用于补充或校验，并缓存结果。
- 两种来源均不可用时，显示“无法确认是否为工作日”。
- 不是工作日时显示“所选择的时间不是工作日，请确认是否要填写一日活动计划”。
- 工作日判断只提供软提示，不阻止填写、保存、生成或导出。

## 10. 教案栏目与格式

所有 AI 结果进入结构化字段，教师均可编辑。系统不能只保存一段不可解析文本作为最终教案。

### 10.1 晨间活动

| 字段 | 规则 |
| --- | --- |
| 体能大循环 | 固定文本，不由 AI 改写 |
| 集体游戏 | 游戏名称使用书名号 |
| 自主游戏 | 游戏名称使用书名号 |
| 重点指导 | 从集体游戏或自主游戏中选择一个 |
| 活动目标 | 3 点，按 `1. 2. 3.` 分行，每点以句号结束 |
| 指导要点 | 3 点，按 `1. 2. 3.` 分行，每点以句号结束 |

### 10.2 晨间谈话

| 字段 | 规则 |
| --- | --- |
| 话题 | 使用书名号展示 |
| 问题设计 | 3 点，按 `1. 2. 3.` 分行，每点以问号结束 |

### 10.3 集体活动

| 字段 | 规则 |
| --- | --- |
| 活动主题 | 中文主题，使用书名号展示 |
| 活动目标 | 原教案内容优先，缺失时 AI 补充，保留编号和换行 |
| 活动准备 | 条目化展示，保留换行 |
| 活动重点 | 原教案缺失时，AI 根据年龄特点补写一句 |
| 活动难点 | 原教案缺失时，AI 根据年龄特点补写一句 |
| 活动过程 | 使用 `一、二、三` 等大环节，并保留 `师：`、`幼：`、`小结：` 等结构 |
| 新增环节 | AI 根据年龄特点新增一个环节，结构化标记，不插入“AI 新增”文字 |

教师修改新增环节正文后，新增标记继续保留。页面提供显式“取消新增标记”操作；只有教师主动取消后，该内容才按普通文字导出。

### 10.4 室内区域游戏

| 字段 | 规则 |
| --- | --- |
| 游戏区域 | 从当前班级室内区域中选择或组合 |
| 重点指导 | 从所选区域中选择一个 |
| 活动目标 | 3 点，编号分行，每点句号结束 |
| 指导要点 | 3 点，编号分行，每点句号结束 |
| 支持策略 | 3 点，编号分行，每点句号结束 |

### 10.5 下午户外游戏

| 字段 | 规则 |
| --- | --- |
| 游戏区域 | 从当前班级户外区域中选择或组合 |
| 重点指导 | 从所选区域中选择一个 |
| 活动目标 | 3 点，编号分行，每点句号结束 |
| 指导要点 | 3 点，编号分行，每点句号结束 |
| 支持策略 | 3 点，编号分行，每点句号结束 |

系统内部统一使用“重点指导”。如果 Word 模板固定文案为“重点观察”，只做字段位置映射，不改变业务术语。

### 10.6 一日活动反思

- 使用 `highlights`、`issues`、`adjustments` 三个字段；三个值分别做 Unicode NFKC 规范化后拼接，按 Unicode 码点计数（内容中的空格和标点计入），合计不超过 200。
- 由 AI 根据前五个上游栏目生成结构化反思，教师可编辑；导出时按三行输出。
- 一键生成不包含反思。前五个上游栏目满足固定结构完整性后，教师可以显式发起生成或重新生成反思。

## 11. 计划管理流程

### 11.1 首页与查询

提供：

- 日历视图。
- 列表视图。
- 按班级筛选。
- 按日期范围筛选。
- 按编写教师筛选。
- 按归档状态筛选。

首期不提供全文搜索。

### 11.2 创建或打开计划

1. 教师选择本人关联班级和活动日期。
2. 系统检查 `(kindergarten_id, class_id, plan_date)`。
3. 已存在时打开原教案，不创建重复记录。
4. 不存在时使用学期、年龄段、班级区域和编写者上下文创建空教案。
5. 系统展示日期、学期和工作日软提示。

### 11.3 保存

- 提供显式“保存”按钮。
- 停止输入约 3 秒后自动保存。
- 页面显示“保存中”“已保存”或“保存失败”。
- 使用版本号乐观锁检测并发修改。
- 发现版本冲突时禁止静默覆盖，并提示用户刷新或比较最新内容。
- 手动保存创建快照；自动保存不创建快照。

### 11.4 归档

- 教案不使用草稿、完成、审核等状态。
- 归档以 `archived_at` 表示。
- 归档后默认只读。
- 任课教师可以归档或恢复本人关联班级教案。
- 管理员可以归档或恢复全园教案。
- 恢复归档后可以继续编辑。
- 首期不提供教案删除功能。

## 12. AI 模型与提示词管理

### 12.1 七个任务级提示词

首期固定 7 个稳定任务标识：

1. 晨间活动。
2. 晨间谈话。
3. 集体活动教案拆分。
4. 集体活动新增环节。
5. 室内区域游戏。
6. 下午户外游戏。
7. 一日活动反思。

提示词中心只管理自然语言文本和变量占位符。输出 Schema、字段结构和校验规则固定在代码中，管理员不能通过提示词修改数据契约。占位符只允许白名单纯替换词法 `\{\{[ \t]*([a-z][a-z0-9_]*)[ \t]*\}\}`；不得执行表达式、过滤器、循环、函数、嵌套字段访问或递归渲染变量值。数组和对象使用稳定 JSON 序列化。

年龄段作为提示词变量传入，首期不创建年龄段专属提示词版本。

### 12.2 系统默认提示词

- 以上 7 个任务都提供只读系统默认提示词。
- 默认内容以旧仓库现有提示词为业务来源，按新系统任务标识、变量白名单、固定 Schema 和当前模板要求重新整理。
- 管理员不能直接编辑系统默认版本。
- 没有已发布自定义版本时使用系统默认。
- 已发布自定义版本优先于系统默认。

### 12.3 自定义提示词生命周期

- 管理员创建和编辑草稿。
- 草稿发布后成为新的已发布版本。
- 业务生成只读取已发布版本。
- 支持查看完整历史版本。
- 支持将历史版本恢复为新的当前已发布版本。
- 修改草稿不会立即影响生产生成。
- 每次生成记录所用提示词标识和版本。

### 12.4 变量白名单

每个任务定义固定变量白名单。至少可以按任务使用：

- 活动日期、星期、教学周和季节。
- 幼儿园、班级和年龄段。
- 室内或户外区域。
- 教师填写的上下文。
- 集体活动原始文本或已结构化字段。
- 当前一日活动计划内容。

引用白名单外变量时禁止保存或发布，并显示中文错误。

### 12.5 提示词测试运行

- 管理员可以对草稿或历史版本执行测试运行。
- 测试使用管理员选择的模型档案和脱敏示例变量。
- 测试结果不写入正式教案。
- 每个提示词保留最近 20 条测试记录。
- 管理员可以清空测试记录。
- 测试记录包含提示词版本、模型档案、变量摘要、状态、耗时和输出结果。
- 测试记录不保存 API Key。

## 13. AI 生成流程

### 13.1 一键生成

- 一键生成晨间活动、晨间谈话、室内区域游戏和下午户外游戏四栏。
- 一键生成不包含集体活动或一日活动反思。
- 每个栏目作为独立后台子任务执行。
- 成功栏目可以立即进入预览，不等待其他栏目。
- 单个栏目失败不回滚成功栏目。
- 教师可以只重试失败栏目。

### 13.2 分栏目生成

- 每个 AI 栏目都提供独立生成或重新生成。
- 新结果先进入预览状态。
- 教师选择“采用新内容”后才替换当前栏目并创建快照。
- 教师选择“保留原内容”、关闭预览或生成失败时，当前栏目保持不变。
- 预览只有目标栏目或该次生成实际使用的输入变化时才失效；无关栏目变化不影响采用。采用时仍使用全局版本号防止写入覆盖。

### 13.3 集体活动

1. 教师粘贴纯文本或上传 `.docx`。
2. 系统提取并展示原始文本。
3. 教师确认后触发集体活动教案拆分。
4. AI 返回主题、目标、准备、重点、难点和活动过程的结构化数据。
5. 原教案缺少重点或难点时，AI 各补写一句。
6. 独立调用新增环节任务，在活动过程中插入一个适龄环节。
7. 新增环节带结构化来源和导出高亮标记。
8. 教师预览并采用后写入当前教案。

`.docx` 上传文件只作临时解析。解析成功后删除临时文件，长期仅保存：

- 提取后的原始文本。
- 原始文件名。
- 上传时间。
- 内容哈希。

首期不长期保存原始上传附件。

### 13.4 输入上下文

AI 仅接收当前任务必需的上下文：

- 活动日期、星期、周次和季节。
- 班级和年龄段。
- 当前班级区域。
- 教师填写的当前栏目上下文。
- 集体活动原始文本或当前计划必要内容。

默认不发送教师账号、手机号、幼儿姓名等身份信息，不发送虚构天气。

### 13.5 异步状态

API 在 2 秒内创建 AI 任务并返回任务 ID。Web 每 1–2 秒短轮询任务状态，并展示：

- 排队中。
- AI 正在生成，请稍候。
- 生成失败，正在重试。
- 等待教师确认。
- 已采用。
- 已保留原内容。
- 最终失败。

### 13.6 重试策略

首次调用失败后最多自动重试两次，总调用次数最多三次。

| 失败类型 | 是否重试 |
| --- | --- |
| 网络错误、超时 | 是 |
| HTTP 429 | 是 |
| HTTP 5xx | 是 |
| JSON 解析或 Schema 校验失败 | 是，并附加格式修正要求 |
| API Key 无效 | 否 |
| 余额不足 | 否 |
| 权限不足 | 否 |
| 模型不存在 | 否 |
| 配置错误 | 否 |

最终失败后保留原内容，显示稳定中文错误，并允许教师手动填写或再次触发该栏目。

## 14. Word 导出

### 14.1 导出前检查

1. 导出前保存当前内容。
2. 保存失败时禁止导出。
3. 检查关键栏目是否缺失。
4. 有缺失时列出缺失栏目并要求教师再次确认。
5. 缺失提示不阻止教师确认后导出，也不自动调用 AI 补全。

关键栏目只包括晨间活动、晨间谈话、集体活动、室内区域游戏和下午户外游戏；一日活动反思为空不触发缺失确认，但模板中的反思三行位置仍须保留。

### 14.2 模板规则

- 只使用 `templates/teacherplan/teacherplan.docx`。
- 基于模板副本填充，禁止覆盖原模板。
- 不实现模板缺失时从零重建表格的降级方案。
- 保持表格结构、固定标签、字体、字号、段落和换行。
- 幼儿园名称从系统设置读取。
- 学期范围、班级和编写教师从教案快照读取。
- 只有带高亮标记的新增集体活动环节使用红色字体。
- 不额外插入“新增”“新增环节”或“AI 新增”等文字。

### 14.3 文件名与存储

教师下载文件名：

```text
一日活动计划_{班级}_{活动日期YYYY-MM-DD}.docx
```

示例：

```text
一日活动计划_小四班_2026-03-04.docx
```

- 班级名称移除文件系统非法字符。
- 教师可见文件名不附加导出时间。
- 服务器使用内部 UUID 或导出记录 ID 作为唯一存储键，避免同名覆盖。
- 文件保存到受保护的持久化导出卷。
- 文件不进入 Git。
- 下载和重新下载必须校验园所、班级和角色权限。

### 14.4 导出历史

- 每次导出保留独立服务器副本和导出记录。
- 历史在教案详情或编辑页展示。
- 教师可以重新下载本人有权访问的历史导出。
- 归档或恢复教案不删除导出历史。
- 服务器副本缺失时显示中文错误，不静默重新生成旧导出。

## 15. 安全、隐私与审计

### 15.1 敏感数据

- 密码使用 Argon2id 哈希。
- AI Key 由服务端加密保存，主加密密钥位于数据库之外。
- 页面只显示脱敏 Key，不回显明文。
- 日志、异常、审计、任务消息和测试快照不得包含 Key、令牌或完整敏感正文。
- 临时上传文件必须在成功、失败或超时后清理。
- 所有园所业务表包含 `kindergarten_id` 和审计时间字段。
- Repository 的读取、写入、更新、归档、唯一性检查均显式使用 `kindergarten_id`。

### 15.2 审计事件

至少记录：

- 登录成功、登录失败和密码修改。
- 账号创建、启停、重置密码和角色变更。
- 班级、学期和区域设置变更。
- 模型档案新增、修改、启停。
- 提示词发布、恢复和测试记录清空。
- AI 任务创建、重试、成功、失败和结果采用。
- 手动保存、归档、恢复归档和版本恢复。
- Word 导出和历史文件下载。

审计记录至少保留 1 年。审计仅保存必要元数据，不保存 API Key、完整教案正文或完整 AI 请求。

## 16. 异常与降级

| 场景 | 产品行为 |
| --- | --- |
| AI 未配置 | 手工编辑可继续，AI 按钮提示先配置模型 |
| AI 调用最终失败 | 保留原内容，标记失败，允许手工填写或重试 |
| 部分栏目失败 | 成功栏目可预览和采用，失败栏目独立重试 |
| 节假日数据不可用 | 显示软提示，继续填写、保存和导出 |
| 自动保存失败 | 显示失败状态，保留页面内容，允许手动重试 |
| 乐观锁冲突 | 禁止覆盖，提示刷新或比较最新内容 |
| Word 模板哈希不符 | 禁止静默导出，记录错误并提示管理员检查模板 |
| Word 生成失败 | 保留已保存教案，不创建成功导出记录 |
| 导出服务器副本缺失 | 显示明确错误，不自动重建历史文件 |
| Redis 或 Worker 不可用 | 数据库事务保存任务后仍返回任务 ID 与 `pending_dispatch`；手工编辑可继续，恢复扫描在依赖恢复后投递任务 |

## 17. 非功能要求

### 17.1 容量与性能

- 支持最多 100 个账号。
- 支持 30 个并发在线会话。
- 普通页面和列表操作在正常网络下 95% 于 2 秒内响应。
- 保存操作 95% 于 1 秒内完成。
- AI 任务在 2 秒内创建并返回任务 ID，不承诺外部模型完成时长。
- 队列至少容纳 100 个待处理任务。
- AI 实际并发数由模型档案限流配置控制。

### 17.2 生产可用性与恢复（延后验收）

以下是未来生产方案必须重新评审的目标，不属于当前功能开发的实现或验收范围：

- PostgreSQL 与持久化导出文件每日备份。
- 备份至少保留 30 天，并加密存放在应用服务器之外。
- 恢复点目标（RPO）不超过 24 小时。
- 恢复时间目标（RTO）不超过 4 小时。
- 数据库迁移和生产升级前额外备份。
- 定期执行恢复演练并记录结果。

### 17.3 兼容性

- 优先支持桌面浏览器和平板。
- 手机端只保证登录、浏览和简单修改。
- 首期仅提供简体中文。
- 所有业务日期按 `Asia/Shanghai` 计算。
- 数据库时间戳统一存储为 UTC。

### 17.4 可测试性

- 默认自动化测试不访问真实 AI、节假日 API 或外部网络。
- AI、时间、节假日、Word 和任务队列使用明确测试替身。
- 关键数据库迁移和隔离行为必须在 PostgreSQL 环境验证，不能只依赖 SQLite。
- Word 测试验证模板未改变、字段位置、换行、中文字体和红字范围。

## 18. 验收标准

### 18.1 账号与权限

- [ ] 首位管理员可以安全初始化。
- [ ] 管理员可以创建、启停账号和重置密码。
- [ ] 不允许停用最后一个有效管理员。
- [ ] 教师不能访问未关联班级教案。
- [ ] 管理员默认不能修改教师正文。
- [ ] 服务端可以阻止直接越权 API 请求。

### 18.2 设置

- [ ] 管理员可以维护幼儿园、学期、班级、教师和模型档案。
- [ ] 任课教师可以维护本人班级室内、户外区域。
- [ ] 区域保持顺序，不接受空项或同类重复项。
- [ ] API Key 加密保存且页面不回显明文。
- [ ] 模型档案可以声明能力并设置默认档案。

### 18.3 教案管理

- [ ] 教师可以手工完成并保存全部结构化栏目。
- [ ] 同班同日不能创建重复教案。
- [ ] 选择已有日期自动打开原教案。
- [ ] 归档教案只读且仍占用唯一键。
- [ ] 自动保存、手动保存和保存状态反馈正确。
- [ ] 并发修改不会静默覆盖。
- [ ] 历史快照可以恢复，恢复前内容仍被保留。
- [ ] 日期创建后不能修改。

### 18.4 日期规则

- [ ] 学期开始日期所在自然周为第一周。
- [ ] 后续每到周一周次递增。
- [ ] 星期日期和中文周次格式正确。
- [ ] 学期外和非工作日只软提示。
- [ ] 节假日服务不可用不阻止主要流程。
- [ ] 季节按固定月份映射，不使用实时天气。

### 18.5 提示词与 AI

- [ ] 7 个任务均有只读系统默认提示词。
- [ ] 管理员可以创建草稿、发布、查看历史和恢复版本。
- [ ] 未发布草稿不影响生产生成。
- [ ] 白名单外变量无法保存或发布。
- [ ] 每个提示词保留最近 20 条测试记录并可清空。
- [ ] 一键生成不包含集体活动。
- [ ] 一键生成不包含一日活动反思；反思只能在前五个上游栏目完整后由教师显式发起。
- [ ] 每个栏目可以独立生成和重新生成。
- [ ] 部分失败不回滚成功栏目。
- [ ] 重新生成必须预览确认，未采用时原内容不变。
- [ ] 可恢复错误和 Schema 错误最多重试两次。
- [ ] 认证、余额、权限、模型和配置错误不重试。

### 18.6 集体活动

- [ ] 支持粘贴文本和上传 `.docx`。
- [ ] 临时文件解析后被清理，原始文本和元数据被保存。
- [ ] AI 可以结构化拆分主题、目标、准备、重点、难点和过程。
- [ ] 原教案缺少重点、难点时可以补充。
- [ ] AI 新增一个适龄活动环节并带结构化标记。
- [ ] 教师编辑新增环节后标记仍保留。
- [ ] 教师可以显式取消新增标记。

### 18.7 Word 导出

- [ ] 导出前保存当前内容，保存失败时禁止导出。
- [ ] 缺失栏目提示不阻止教师确认后导出。
- [ ] 导出严格使用当前固定模板，原模板文件未改变。
- [ ] 学期、班级、教师及各栏目写入正确位置。
- [ ] 只有带标记的新增环节正文为红色。
- [ ] 文件名为 `一日活动计划_{班级}_{YYYY-MM-DD}.docx`。
- [ ] 重复导出不覆盖服务器副本。
- [ ] 导出历史记录教案版本和模板哈希。
- [ ] 有权限的用户可以重新下载历史文件。

### 18.8 运维与安全

- [ ] 审计事件完整且不包含敏感正文或密钥。
- [ ] 审计记录保留至少 1 年。
- [ ] 常规自动化测试不访问真实外部服务。

数据库与导出备份恢复、RPO/RTO 和演练验收延后到全部功能完成后，由 ADR-0009 规定的生产部署复审定义。

## 19. 风险与缓解

| 风险 | 缓解措施 |
| --- | --- |
| AI 输出格式波动 | 固定 Schema、校验、修正重试、教师预览 |
| AI 覆盖人工内容 | 预览采用、乐观锁、关键快照 |
| Word 模板发生变化 | 哈希校验、固定映射、模板回归测试 |
| 队列任务重复执行 | 幂等键、状态机、数据库约束和版本检查 |
| 外部节假日服务波动 | 本地库、缓存和软提示降级 |
| 园所或班级越权 | API 鉴权、Repository 隔离和权限测试 |
| SQLite 测试掩盖 PostgreSQL 问题 | PostgreSQL 迁移与关键集成测试 |
| 导出文件泄露 | 私有持久化卷、鉴权下载、备份加密 |
| 文档与两个实现分支漂移 | 独立文档提交、cherry-pick 和分支状态记录 |

## 20. 实施依赖

本 PRD 对应功能按以下依赖顺序交付：

1. 工程骨架与 CI。
2. 认证授权。
3. 必要设置。
4. AI 模型档案与提示词管理。
5. 手工教案闭环。
6. 异步 AI 生成。
7. Word 导出和导出历史。
8. 功能、权限、数据隔离和安全行为验收。
9. 所有功能完成后，另行复审生产访问网络、部署、备份与恢复方案。

Codex 与 Trae 可以采用不同内部实现，但不得改变上述产品依赖、业务不变量和验收结果。

## 21. 架构设计引用

后台任务统一采用 Dramatiq 2.x + Redis，任务状态使用 1–2 秒短轮询。具体服务边界与可靠性约束见 [`docs/design/system-architecture.md`](../design/system-architecture.md)。根据 ADR-0009，反向代理、生产访问网络、部署拓扑和主密钥托管实现延后到全部功能完成后复审。

架构实现不得改变本 PRD 已定义的用户行为、安全结果和验收标准。
