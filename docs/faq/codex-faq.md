## 4. Codex 审计结论与用户确认

### Q1：Word 模板是否包含真实个人信息？

是。用户确认模板本体中的教师姓名属于真实人员姓名。本文不得复述该姓名。

用户已确认的处理方向：

1. 用明确的虚构占位名称“示例教师”替换当前模板中的真实姓名，同时保持表格、字体、字号、段落和版式不变。
2. 从 Git 历史中彻底清除该真实姓名；不能只删除当前版本。
3. Git 历史改写和远程强制更新必须作为独立高风险步骤执行，不得与普通文档修改混在一起。
4. 尚未执行模板修改、历史改写或远程更新。

影响：模板内容变化会改变 SHA-256。最终修订必须重新计算哈希，并同步规格、计划、任务、测试要求及其他所有固定哈希引用。

### Q2：canonical 数据模型与 Spec Kit 冲突时以哪套为准？

用户确认：以后续形成的 Spec Kit 规格、OpenAPI、任务状态机和 `tasks.md` 新版模型为准，再反向同步 canonical 数据模型与数据库 Schema。

已确认的新版语义至少包括：

- `ai.batch` 父任务不投递、不持有租约，也不保存独立执行状态或数据库 attempt；API 的 attempt 投影为 `0/0`，展示状态由恰好四个子任务实时派生。
- 客户端幂等作用域包含园所、操作者、HTTP 方法、规范化路由模板和 key；请求摘要还包含实际 path、业务 query 和 canonical JSON body。
- `ai.batch` 内部子任务不伪造客户端幂等键，使用父任务、栏目等数据库约束防止重复。
- 提示词测试在受理时冻结实际输入、提示词正文与哈希、结果 Schema、非密钥模型调用配置和 `call_config_revision`。
- 地址、模型名、能力或 API Key 的业务变化递增 `call_config_revision`；旧测试任务必须以 `prompt.configuration_changed` 零外部调用失败。
- 每个执行型 AI 任务受理时创建唯一 pending 结果占位，冻结目标栏目基线、实际输入、模型、提示词和结果 Schema。
- Word 导出记录保存不可变的上下文、正文、Schema 版本和正文哈希；Worker 不得执行时重读后来变化的当前教案。

canonical 漂移的主要位置：

- `docs/design/data-model.md` 的 `ai_model_profiles`、`prompt_test_runs`、`background_jobs`、`ai_generation_results` 和 `daily_activity_plan_exports`。
- `docs/design/database-schema.md` 中相同表的物理字段、非空约束、唯一约束、组合外键和状态检查。

尚未修改这些文件。

### Q3：模板说明与新版 AI 流程冲突时以哪套为准？

用户确认：以新版规格为准，更新 Markdown 模板说明，但不改变 `.docx` 版式。

最终说明应明确：

- 集体活动分为两个独立阶段：先拆分/补全并由教师采用保存，再独立生成、预览和采用一个新增环节。
- 新增环节失败不得回滚已采用的拆分结果。
- 反思只读取晨间、晨谈、集体、室内和户外五个上游栏目，不读取旧反思。
- 反思固定为 `highlights`、`issues`、`adjustments` 三个字段，合计不超过 200 个 Unicode 码点，并在 Word 中按三行填写。
- 在线节假日地址使用已确认的 HTTPS 形式，不保留模板说明中的 HTTP 地址。

## 5. Codex 与 Trae 问题的最终结论

本节合并原 `codex-question.md` 的五项问题，并补充后续 Trae 问题的确认结果。选项编号只属于提问时的临时标记，以下完整语义才是最终结论。

### 5.1 ADR-0001 文件命名

- ADR-0001 原文件已重命名为 `docs/ADR/ADR-0001-cloud-only.md`。
- 已同步 ADR 索引、Markdown 链接和其他活跃文档路径引用。
- 这只是命名一致性修订，不改变 ADR 内容、编号或接受状态。

### 5.2 学期自然周与首周边界

- 自然周从周一开始，到周日结束。
- 学期开始日期所在自然周为第一周，此后每到下一个周一递增一周。
- 如果学期从周三开始，周三至周日属于第一周；此前周一、周二仍在学期外，教学周次及显示文本为空。

### 5.3 开发环境主加密密钥与 AI Key 加密

- 开发环境从仓库外、受文件权限保护的本地密钥文件读取主加密密钥。
- 自动化测试使用明确的测试适配器，不读取真实密钥。
- 主密钥不得进入代码库、数据库、镜像、普通环境变量、日志、异常、审计或测试快照。
- 生产密钥托管方案延后至 M9。
- AI Key 使用 AES-256-GCM；每次加密使用随机 96 位 nonce，AAD 绑定园所 ID、模型档案 ID 和 envelope version，并通过 `key_id` 支持轮换。

### 5.4 WCAG 2.2 AA 验收范围

- 使用 WCAG 2.2 AA，不使用原问题稿误写的 WCAG 2.1 AA。
- 覆盖登录、必要设置、教案创建/编辑/保存、AI 预览采用和导出等核心流程。
- 按现有规格执行自动可访问性扫描和完整键盘人工清单，不扩展为首期所有非核心页面的逐页验收承诺。

### 5.5 自动保存失败

- 自动保存失败后不进行后台自动重试。
- 页面保留未保存内容并明确显示失败状态，允许用户手动重试。
- 用户继续编辑后可以触发下一次正常自动保存。
- 版本冲突、权限错误或校验错误不得通过隐藏重试绕过。

### 5.6 历史恢复快照原因

- 历史恢复前先为当前内容创建 `before_restore` 快照。
- 将选定历史复制为新的当前版本后创建 `restored` 快照。
- FR-031 应补充原因映射，并与已经描述恢复步骤的 FR-032、数据模型和数据库约束保持一致。

### 5.7 AI 结果保留与审计追溯

- 等待确认、被放弃或最终失败的完整 AI 结果最多保留 30 天；已采用结果可以立即清除完整副本。
- 清理只移除完整输出和不再需要的详细输入，不删除结果、任务或审计的关联身份。
- 继续保留 `job_id`、状态、模型与提示词版本、Schema、结果哈希、必要操作者和时间元数据。
- 审计至少保留一年，但不得复制完整教案或 AI 输入输出。

### 5.8 后台任务关系与状态

- `parent_job_id` 使用可空同园组合自外键 `(kindergarten_id, parent_job_id) -> background_jobs(kindergarten_id, id)`，删除策略为 `RESTRICT`。
- `parent_job_id` 只表达 `ai.batch` 父子关系；`retry_of_job_id` 使用独立的可空同园组合自外键表达显式重试谱系。
- 禁止自引用；同一父任务的栏目子任务由 `(kindergarten_id, parent_job_id, target_section)` 唯一约束防止重复。
- 在系统架构状态机补充 `awaiting_confirmation -> expired`；`expired` 是等待确认 AI 预览的业务终态，不修改教案正文或创建快照。

### 5.9 测试策略

- 并发、数据库/Redis 故障窗口、重复投递、Worker 崩溃、租约过期和恢复扫描已经在规格、系统架构与任务清单中覆盖。
- 不在数据模型文档中复制详细测试步骤；数据模型只保留并发、幂等和恢复不变量。
- 常规测试使用确定性故障注入，不要求制造真实操作系统级网络分区。
- 生产备份恢复、RPO/RTO 和灾难恢复演练继续延后至 M9。

### 5.10 API 与前端信息架构

- 不在 `docs/design/` 新建重复的完整 API 端点文档；OpenAPI 与 `contracts/README.md` 是 API 契约事实来源，设计文档只增加导航链接。
- 不新建重复的前端信息架构文档；页面范围以规格为准，现有 research 页面信息架构与任务清单提供设计和实施索引。
- 浏览器不得提交 `kindergarten_id` 作为授权依据；API 从服务端会话取得园所 ID，并在 Service、Repository 和数据库约束中显式执行园所隔离。

### 5.11 集体活动新增环节插入位置

- AI 返回单一结构化环节和 `suggested_insert_index`，合法范围为 `0 <= index <= len(process)`。
- 越界索引属于结构错误并进入既定 AI 自动重试，不得截断、改为末尾或自行猜测。
- 结果只作为预览；教师明确采用且版本与生成基线仍一致时，系统按有效索引插入并设置 `is_ai_added=true`。
- 当前内容已经变化时返回冲突，不把旧索引应用到新内容。
