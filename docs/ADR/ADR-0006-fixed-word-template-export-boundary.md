# ADR-0006：一日活动计划采用固定 Word 模板导出边界

日期：2026-07-11

状态：已接受

## 背景

幼儿园已有固定的 Word 表格版式，首期必须稳定导出 `.docx`。旧系统经验表明，直接修改原模板、依赖自然语言整段填充或把 Word 操作放入 UI，容易造成表格、字体、段落和红字范围回归。当前并没有模板上传、模板市场或在线排版需求。

## 决策

1. 一日活动计划只使用仓库中的固定模板 `templates/teacherplan/teacherplan.docx`。
2. 模板按固定表格、单元格和段落映射填充，不建设通用模板引擎。
3. Word 导出是独立适配器并由 Worker 后台执行；业务服务只提供结构化导出上下文。
4. 每次从只读模板副本生成，禁止覆盖原模板。
5. 仅 AI 新增的集体活动环节标红；新增标记保存在结构化数据中，教师显式取消后才转为普通文字。
6. 每次导出保存独立记录和服务器私有副本，记录模板文件名与 SHA-256；内部 UUID 存储键防止覆盖。
7. API 通过权限校验后的下载接口返回文件，不暴露服务器真实路径。
8. 首期只导出 `.docx`，不提供 PDF、在线打印排版、模板上传或模板在线编辑。

## 备选方案

### 通用模板平台

不采用。当前只有一个已确认版式，通用占位符、模板版本选择和在线编辑会显著扩大范围。

### 从零生成 Word 排版

不采用。难以稳定复现园所现有表格和段落样式。

### UI 直接修改模板并下载

不采用。会把页面、文件系统和模板细节耦合，并难以保证审计和导出历史。

### 只向浏览器返回临时文件

不采用。无法重新下载、审计和追溯模板版本，也不能满足备份要求。

## 后果

正面影响：

- 导出版式可预测并可做精确回归测试。
- Word 细节与教案业务和 UI 解耦。
- 重复导出可追溯且不会互相覆盖。

代价与风险：

- 模板结构变化需要同步更新字段映射与测试。
- API 与 Worker 需要共享私有导出卷。
- 固定位置映射比通用占位符更依赖模板哈希和结构检查。

## 实施约束

- 模板目录只读挂载给 Worker。
- 测试覆盖模板哈希、表格结构、字段位置、换行、字体和红字范围。
- 教师可见文件名为 `一日活动计划_班级_日期.docx`；内部文件名必须唯一。
- 导出不完整教案前给出软确认，但必须先成功保存当前内容。

## 复审触发条件

只有出现第二个真实模板、园所自定义模板或 PDF 合规交付需求时，才新增 ADR 评估模板版本与转换能力。
